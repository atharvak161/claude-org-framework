Decision protocol · mandatory multi-perspective consultation
⬡ Multi-Perspective Consultation — Mandatory Steps
  1. 01. Task received by department director — scope confirmed, ownership assigned.
  2. 02. Director assigns the task to 2 or more agents with different focus areas (e.g. one for security implications, one for performance, one for implementation). Each agent works independently — no coordination between them at this stage.
  3. 03. Each agent produces an independent proposal: approach, rationale, trade-offs, risks. Proposals are written to a shared consultation record file.
  4. 04. Director reads all proposals in the consultation record. Compares approaches. May ask agents to respond to each other's proposals (one round of cross-review). No groupthink — dissent is recorded.
  5. 05. Director makes the final decision. Writes the decision, chosen approach, and full rationale to DECISIONS.md before any implementation begins.
  6. 06. Output passes up the review chain: sub-agent output → manager review → director / VP sign-off → Chief of Staff approval → Atharva delivery.
Non-negotiable: no single-agent decisions on consequential matters. Skipping consultation without documented justification is a protocol violation logged to BLOCKERS.md. The Chief of Staff may override any department decision if cross-dept impact is undisclosed.
