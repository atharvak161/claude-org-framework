# 08.3 — Manual Assessment Checklist by Service

## What This Note Covers

The definitive manual assessment checklist for every major cloud service — what to check, what commands to run, and what constitutes a finding. Use this after automated scanning to verify, expand, and validate.

---

## How to Use This Checklist

Automated tools (ScoutSuite, Prowler) catch known misconfiguration patterns against rulesets. Manual review catches:
- Business logic issues (why does this role need these permissions for this specific function?)
- Chained attack paths (misconfiguration A + misconfiguration B = critical finding)
- Context-specific issues (dev credentials in prod environment)
- Novel attack patterns the tools haven't been updated for

Work through each service section in order. Mark findings with severity.

## IAM Checklist

```bash
# Account-level checks
aws iam get-account-summary          # overall account stats
aws iam get-account-password-policy  # password requirements
aws iam list-virtual-mfa-devices     # who has MFA
aws iam generate-credential-report   # CSV of all users + credential status
aws iam get-credential-report        # download the report

# From the credential report, check:
# - user.password_enabled = true AND user.mfa_active = false → NO MFA (finding)
# - user.access_key_1_active = true AND user.access_key_1_last_rotated > 90 days → STALE KEY
# - user.password_last_used = N/A AND user.access_key_*_last_used_date = N/A → UNUSED USER
```

**IAM Checks:**

- [ ] Root account has MFA enabled
- [ ] Root account has NO access keys (`aws iam get-account-summary | grep RootAccessKeys`)
- [ ] All console users have MFA enabled
- [ ] Access keys rotated within 90 days
- [ ] No inline policies (prefer managed)
- [ ] No customer policies with `Action: "*"` on `Resource: "*"`
- [ ] No `AdministratorAccess` on non-admin users
- [ ] Service roles follow least privilege (not `AdministratorAccess`)
- [ ] Trust policies don't allow `Principal: "*"`
- [ ] Permission boundaries used where appropriate
- [ ] No unused IAM users (no login or API activity > 90 days)
- [ ] IAM Access Analyzer enabled in all regions

```bash
# Find policies with admin wildcards
aws iam list-policies --scope Local \
  --query 'Policies[].Arn' --output text | \
  tr '\t' '\n' | while read arn; do
    ver=$(aws iam get-policy --policy-arn $arn --query 'Policy.DefaultVersionId' --output text)
    doc=$(aws iam get-policy-version --policy-arn $arn --version-id $ver \
      --query 'PolicyVersion.Document' --output json)
    echo $doc | python3 -c "
import sys,json
d=json.load(sys.stdin)
for s in d.get('Statement',[]):
    a=s.get('Action','')
    r=s.get('Resource','')
    if (a=='*' or a==['*']) and (r=='*' or r==['*']):
        print('ADMIN WILDCARD in $arn')
" 2>/dev/null
  done
```

## S3 Checklist

- [ ] Account-level "Block Public Access" enabled
- [ ] No publicly accessible buckets (ACL or policy)
- [ ] All buckets have server-side encryption enabled
- [ ] Sensitive buckets have versioning + MFA delete
- [ ] Access logging enabled for sensitive buckets
- [ ] Lifecycle policies configured (data retention)
- [ ] No cross-account access without explicit approval
- [ ] Terraform state buckets are private with strict IAM

```bash
# Account-level block public access
aws s3control get-public-access-block --account-id ACCOUNT_ID

# Per-bucket checks
for BUCKET in $(aws s3 ls | awk '{print $3}'); do
  echo "=== $BUCKET ==="
  
  # Block public access
  aws s3api get-public-access-block --bucket $BUCKET 2>/dev/null || echo "No block public access set"
  
  # ACL
  aws s3api get-bucket-acl --bucket $BUCKET \
    --query 'Grants[?Grantee.URI==`http://acs.amazonaws.com/groups/global/AllUsers`]' 2>/dev/null
  
  # Policy
  aws s3api get-bucket-policy --bucket $BUCKET 2>/dev/null | \
    python3 -c "import sys,json; p=json.load(sys.stdin); print(p.get('Policy',''))" | \
    python3 -c "import sys,json; d=json.loads(sys.stdin.read()); \
      [print('PUBLIC POLICY') for s in d.get('Statement',[]) if s.get('Principal')=='*']"
  
  # Encryption
  aws s3api get-bucket-encryption --bucket $BUCKET 2>/dev/null || echo "NO ENCRYPTION"
  
  # Logging
  aws s3api get-bucket-logging --bucket $BUCKET
done
```

## EC2 and Networking Checklist

- [ ] No security groups with `0.0.0.0/0` on SSH (22) or RDP (3389)
- [ ] No security groups with `0.0.0.0/0` on database ports (3306, 5432, 27017, 6379)
- [ ] No security groups with all ports open (`0-65535`)
- [ ] IMDSv2 required on all instances
- [ ] EBS volumes encrypted
- [ ] No publicly shared EBS snapshots
- [ ] No publicly shared AMIs with sensitive data
- [ ] Default VPC not used for production workloads
- [ ] VPC Flow Logs enabled
- [ ] EC2 instances not using root volume without encryption

```bash
# Security groups with internet access
aws ec2 describe-security-groups \
  --filters "Name=ip-permission.cidr,Values=0.0.0.0/0" \
  --query 'SecurityGroups[].{ID:GroupId,Name:GroupName,Rules:IpPermissions[?contains(IpRanges[].CidrIp,`0.0.0.0/0`)].[{From:FromPort,To:ToPort}][]}' \
  --output json

# IMDSv2 status
aws ec2 describe-instances \
  --query 'Reservations[].Instances[].{ID:InstanceId,IMDSv2:MetadataOptions.HttpTokens}' \
  --output table
# "optional" → IMDSv2 not required (finding)

# Public snapshots
aws ec2 describe-snapshots --owner-ids ACCOUNT_ID \
  --query 'Snapshots[?Public==`true`].{ID:SnapshotId,Desc:Description}'

# Unencrypted EBS volumes
aws ec2 describe-volumes \
  --query 'Volumes[?Encrypted==`false`].{ID:VolumeId,State:State,Size:Size}'
```

## RDS Checklist

- [ ] No RDS instances publicly accessible
- [ ] All RDS instances encrypted at rest
- [ ] No public RDS snapshots
- [ ] Automated backups enabled
- [ ] Multi-AZ enabled (high availability)
- [ ] Minor version auto-upgrade enabled
- [ ] Deletion protection enabled
- [ ] DB parameter groups reviewed (no insecure settings)
- [ ] RDS in private subnet, not default VPC

```bash
# Publicly accessible RDS
aws rds describe-db-instances \
  --query 'DBInstances[?PubliclyAccessible==`true`].{ID:DBInstanceIdentifier,Endpoint:Endpoint.Address,Engine:Engine}'

# Unencrypted RDS
aws rds describe-db-instances \
  --query 'DBInstances[?StorageEncrypted==`false`].{ID:DBInstanceIdentifier,Engine:Engine}'

# Public snapshots
aws rds describe-db-snapshots --include-public \
  --query 'DBSnapshots[?Public==`true`].{ID:DBSnapshotIdentifier,Owner:OwnerName}'

# No backup
aws rds describe-db-instances \
  --query 'DBInstances[?BackupRetentionPeriod==`0`].DBInstanceIdentifier'
```

## Lambda Checklist

- [ ] No Lambda function URLs with `AuthType: NONE`
- [ ] No functions with wildcard execution roles
- [ ] No secrets in environment variables
- [ ] Function code reviewed for injection vulnerabilities
- [ ] No outdated runtimes (check EOL dates)
- [ ] Resource policies reviewed (who can invoke)
- [ ] VPC-connected functions (if they need internal DB access)

```bash
# Functions with public URLs (no auth)
aws lambda list-functions --region REGION \
  --query 'Functions[].FunctionName' --output text | tr '\t' '\n' | \
  while read fn; do
    auth=$(aws lambda list-function-url-configs --function-name $fn --region REGION \
      --query 'FunctionUrlConfigs[].AuthType' --output text 2>/dev/null)
    [ "$auth" = "NONE" ] && echo "PUBLIC URL (no auth): $fn"
  done

# Functions with admin execution roles
aws lambda list-functions --region REGION \
  --query 'Functions[].{Name:FunctionName,Role:Role}' --output table

# Environment variables scan
aws lambda list-functions --region REGION \
  --query 'Functions[].FunctionName' --output text | tr '\t' '\n' | \
  while read fn; do
    env=$(aws lambda get-function-configuration --function-name $fn --region REGION \
      --query 'Environment.Variables' --output json 2>/dev/null)
    echo $env | grep -qiE "password|secret|key|token" && echo "POTENTIAL SECRET IN ENV: $fn"
  done
```

## CloudTrail Checklist

- [ ] Multi-region trail enabled
- [ ] Trail logging is active (`IsLogging: true`)
- [ ] Log file validation enabled
- [ ] CloudTrail logs sent to CloudWatch Logs
- [ ] CloudWatch alarms on security events (root login, IAM changes, failed auth)
- [ ] Trail logs stored in S3 with MFA delete
- [ ] Data events enabled for sensitive S3 buckets
- [ ] No gaps in regional coverage

```bash
# List all trails and status
aws cloudtrail describe-trails --include-shadow-trails \
  --query 'trailList[].{Name:Name,MultiRegion:IsMultiRegionTrail,LogGroup:CloudWatchLogsLogGroupArn}'

# Check logging status for each trail
for TRAIL in $(aws cloudtrail describe-trails --query 'trailList[].TrailARN' --output text); do
  STATUS=$(aws cloudtrail get-trail-status --name $TRAIL \
    --query '{Logging:IsLogging,LatestDelivery:LatestDeliveryTime}')
  echo "$TRAIL: $STATUS"
done

# Check data events
aws cloudtrail get-event-selectors --trail-name TRAIL_NAME
# DataResources empty → no S3 data event logging

# Check CloudWatch metric filters for security alerts
aws cloudwatch describe-alarms --alarm-names \
  UnauthorisedAPICalls RootAccountUsage IAMPolicyChanges
```

## VPC Checklist

- [ ] VPC Flow Logs enabled for all VPCs
- [ ] No overly permissive NACLs
- [ ] Default security group has no inbound/outbound rules
- [ ] VPC peering connections reviewed (no unexpected peers)
- [ ] No internet gateway attached to production VPCs with no NAT
- [ ] S3 and DynamoDB gateway endpoints in use (keep traffic off internet)

```bash
# VPC Flow Logs status
aws ec2 describe-vpcs --query 'Vpcs[].VpcId' --output text | tr '\t' '\n' | \
  while read vpc; do
    fl=$(aws ec2 describe-flow-logs \
      --filter Name=resource-id,Values=$vpc \
      --query 'FlowLogs[].FlowLogStatus' --output text)
    [ -z "$fl" ] && echo "NO FLOW LOGS: $vpc" || echo "Flow logs $fl: $vpc"
  done

# Default security group check
aws ec2 describe-security-groups \
  --filters "Name=group-name,Values=default" \
  --query 'SecurityGroups[].{VPC:VpcId,Ingress:IpPermissions,Egress:IpPermissionsEgress}'
```

## Secrets Manager / Parameter Store Checklist

- [ ] All secrets have rotation enabled
- [ ] Secrets not duplicated between Parameter Store and Secrets Manager
- [ ] No secrets with `ResourcePolicy` allowing broad access
- [ ] KMS key used for Parameter Store SecureString (not default key if compliance required)
- [ ] Unused secrets identified and deleted

```bash
# Secrets without rotation
aws secretsmanager list-secrets \
  --query 'SecretList[?RotationEnabled==`false`].{Name:Name,LastRotated:LastRotatedDate}'

# All secrets — enumerate values if accessible
aws secretsmanager list-secrets --query 'SecretList[].{Name:Name,ARN:ARN}' --output table
```
