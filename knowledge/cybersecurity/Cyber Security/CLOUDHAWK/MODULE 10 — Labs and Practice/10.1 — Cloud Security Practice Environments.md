# 10.1 — Cloud Security Practice Environments

## What This Note Covers

Every intentionally vulnerable cloud lab available for practising the techniques in this curriculum — how to deploy them, what they cover, and which order to work through them for maximum skill development.

---

## The Practice Problem

Cloud security skills cannot be developed by reading alone. Every technique in this curriculum requires hands-on repetition against real cloud APIs. The environments in this note provide legal, isolated targets for exactly that.

**Principle:** Never practice offensive cloud techniques against real accounts you don't own. These labs exist so you don't need to.

---

## Tier 1 — Zero-Cost, No Account Required

### flaws.cloud (AWS)

Created by Scott Piper. The original cloud security CTF. Each level is a real misconfigured AWS resource. You solve it by finding the secret that unlocks the next level.

```
URL: http://flaws.cloud
Cost: Free
Account needed: AWS CLI (your own), no AWS account required for the first levels
Topics:
  Level 1: Public S3 bucket enumeration
  Level 2: Authenticated S3 access with any AWS credentials
  Level 3: S3 bucket with leaked access keys in object content
  Level 4: EC2 snapshot exposure
  Level 5: EC2 metadata service (instance role credential theft)
  Level 6: Security misconfiguration requiring IAM enumeration

Getting started:
  # Level 1 hint: the answer is in DNS
  nslookup flaws.cloud
  aws s3 ls s3://flaws.cloud --no-sign-request
```

### flaws2.cloud (AWS)

Sequel to flaws.cloud with two parallel tracks.

```
URL: http://flaws2.cloud
Cost: Free
Account needed: AWS CLI

Attacker path:
  Level 1: Container registry exploitation
  Level 2: Lambda function exposure
  Level 3: IAM enumeration and role assumption

Defender path (work through AFTER attacker path):
  Uses CloudTrail logs from the attacker path scenarios
  Answer forensic questions: "What IP did the attacker come from?"
  "What IAM actions were taken?" → blue team mindset development
```

## Tier 2 — Requires Your Own Cloud Account (Low Cost)

### CloudGoat (AWS) — Best AWS Practice Environment

By Rhino Security Labs. Deploys real vulnerable AWS infrastructure (using Terraform) into your own AWS account. Scenario-based — each one simulates a realistic attack chain.

```
URL: https://github.com/RhinoSecurityLabs/cloudgoat
Cost: Small AWS charges (~$1-5 per session)
Account needed: Your own AWS account

Install:
  git clone https://github.com/RhinoSecurityLabs/cloudgoat
  cd cloudgoat
  pip3 install -r requirements.txt
  ./cloudgoat.py config profile default

Deploy a scenario:
  ./cloudgoat.py create SCENARIO_NAME

Destroy when done:
  ./cloudgoat.py destroy SCENARIO_NAME
```

**Scenarios — work in this order:**

```
BEGINNER:
  cloud_breach_s3
    → Public S3 bucket → escalate to EC2 → steal instance profile → full admin
    Techniques: S3 enum, instance profile abuse, IAM enumeration

  ec2_ssrf
    → Web app with SSRF → steal EC2 metadata credentials → escalate
    Techniques: SSRF, metadata service, IAM escalation

  iam_privesc_by_rollback
    → Low-privilege user → roll back IAM policy version → admin
    Techniques: CreatePolicyVersion, SetDefaultPolicyVersion

INTERMEDIATE:
  lambda_privesc
    → Lambda function abuse → iam:PassRole escalation → admin
    Techniques: Lambda execution role, PassRole chain

  vulnerable_cognito
    → Cognito misconfiguration → identity pool abuse → AWS access
    Techniques: Cognito, identity federation

  sqs_flag_shop
    → SQS business logic abuse → manipulate message content → get flag
    Techniques: SQS injection, business logic

ADVANCED:
  codebuild_secrets
    → CI/CD pipeline secret leakage → cross-account access
    Techniques: CodeBuild env vars, cross-account

  detection_evasion
    → Achieve objectives while avoiding GuardDuty detection
    Techniques: Evasion, CloudTrail analysis

  ecs_takeover
    → ECS task exploitation → container escape → host compromise
    Techniques: Container, ECS metadata, task role abuse
```

### AzureGoat (Azure)

By INE Labs. Intentionally vulnerable Azure environment deployed via Terraform.

```
URL: https://github.com/ine-labs/AzureGoat
Cost: Azure charges — use free credit ($200 for new accounts)
Account needed: Azure subscription

Install:
  git clone https://github.com/ine-labs/AzureGoat
  cd AzureGoat
  az login
  terraform init
  terraform apply

Topics covered:
  - Azure AD misconfiguration
  - Managed identity abuse
  - Storage account public access
  - App Service credential leakage
  - Key Vault access misconfiguration
  - Function App privilege escalation
  - SQL injection via Azure services

Destroy:
  terraform destroy
```

### GCPGoat (GCP)

By INE Labs. Equivalent of AzureGoat for GCP.

```
URL: https://github.com/ine-labs/GCPGoat
Cost: GCP charges — use free $300 trial credit
Account needed: GCP project

Install:
  git clone https://github.com/ine-labs/GCPGoat
  cd GCPGoat
  gcloud auth application-default login
  terraform init
  terraform apply

Topics covered:
  - Service account key exposure via public GCS bucket
  - Cloud Function with overprivileged SA
  - GCS bucket misconfiguration
  - IAM privilege escalation via custom role
  - Metadata service credential theft
  - Kubernetes workload identity abuse

Destroy:
  terraform destroy
```

## Tier 3 — Commercial Labs (HackTheBox, TryHackMe)

### HackTheBox — Cloud Machines and Challenges

```
URL: https://hackthebox.com
Cost: Free tier (retired machines) + Pro subscription ~£14/month

Cloud-specific content:
  - Cloud category machines (AWS, Azure, GCP themed)
  - Cloud challenges (S3 enumeration, IAM privilege escalation)
  - Dedicated AWS/Azure paths in Pro Labs

Recommended approach:
  1. Start with "Cloud" tagged easy/medium machines
  2. Progress to hard machines after CloudGoat
  3. Pro Labs for full enterprise simulation
```

### TryHackMe — Guided Cloud Rooms

```
URL: https://tryhackme.com
Cost: Free tier + Premium ~£10/month

Notable cloud rooms:
  - "AWS" path
  - "Cloud Security" learning path
  - Individual rooms: S3, IAM, cloud basics

Best for: Complete beginners before moving to CloudGoat
Limitation: More guided than CloudGoat — less true independent hacking
```

## Recommended Learning Path — Sequenced

```
PHASE 1 — Foundations (Weeks 1-2)
  1. Read CLOUDHAWK MODULE 0 thoroughly
  2. Set up AWS free tier account + configure CLI
  3. Play flaws.cloud levels 1-3 (public S3, authenticated access)
  4. Read MODULE 1 (AWS IAM fundamentals through S3 attacks)

PHASE 2 — First Exploitation (Weeks 3-4)
  5. flaws.cloud levels 4-6 (snapshots, metadata, IAM)
  6. CloudGoat: cloud_breach_s3
  7. CloudGoat: ec2_ssrf
  8. flaws2.cloud attacker path

PHASE 3 — IAM Mastery (Weeks 5-6)
  9. CloudGoat: iam_privesc_by_rollback
  10. CloudGoat: lambda_privesc
  11. Read MODULE 5 (serverless) + MODULE 6 (lateral movement)
  12. flaws2.cloud defender path

PHASE 4 — Azure and GCP (Weeks 7-8)
  13. Set up Azure free account
  14. AzureGoat — work through all scenarios
  15. Read MODULE 2 (Azure) + MODULE 3 (GCP)
  16. Set up GCP free account
  17. GCPGoat — work through all scenarios

PHASE 5 — Advanced (Weeks 9-12)
  18. CloudGoat: remaining advanced scenarios
  19. HTB cloud machines
  20. MODULE 4 (containers) + MODULE 7 (recon) + MODULE 8 (methodology)
  21. Run a full mock assessment against your own cloud accounts

PHASE 6 — Bug Bounty / Real Engagements
  22. Join HackerOne/Bugcrowd — target cloud-heavy programs
  23. Read MODULE 9 (persistence) — use defensively to understand defender perspective
  24. Build your own cloud security home lab environment
```

```bash
./cloudgoat.py create ec2_ssrf
./cloudgoat.py create iam_privesc_by_rollback

terraform init && terraform apply

# GCPGoat
git clone https://github.com/ine-labs/GCPGoat && cd GCPGoat
gcloud auth application-default login
terraform init && terraform apply
terraform destroy  # cleanup
```
