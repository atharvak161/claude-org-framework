# 10.2 — Cloud Security Certifications Roadmap

## What This Note Covers

The cloud security certification landscape — which certs matter, in what order to pursue them, what each actually covers, and how they map to your current background and career goals.

---

## The Certification Landscape — Overview

Cloud security certifications fall into three tiers:

```
TIER 1 — Foundation (get these first if you don't have cloud fundamentals)
  AWS Solutions Architect Associate (SAA-C03)
  Microsoft Azure Fundamentals (AZ-900)
  Google Cloud Associate Cloud Engineer

TIER 2 — Security-Specific (your primary targets)
  AWS Certified Security — Specialty (SCS-C02)
  Microsoft Certified: Azure Security Engineer Associate (AZ-500)
  Google Professional Cloud Security Engineer
  CCSP (Certified Cloud Security Professional — ISC2)

TIER 3 — Advanced / Offensive (after professional experience)
  PNPT (Practical Network Penetration Tester) — has cloud sections
  OSCP — not cloud-specific but respected by employers
  Specific vendor red team certs (AWS has none official — industry-recognised)
```

---

## AWS Certified Security — Specialty (SCS-C02)

**The most important cloud security cert for AWS-focused roles.**

```
Exam code:     SCS-C02
Duration:      170 minutes
Questions:     65 (multiple choice + multiple response)
Cost:          ~$300 USD
Validity:      3 years
Prerequisites: Recommended: 5+ years IT security + 2+ years AWS hands-on
               (Practical: AWS SAA + 6 months hands-on is sufficient)

Domain breakdown:
  Domain 1: Threat Detection and Incident Response (14%)
    → GuardDuty, CloudTrail analysis, incident response playbooks
  Domain 2: Security Logging and Monitoring (18%)
    → CloudTrail, CloudWatch, Security Hub, Config rules
  Domain 3: Infrastructure Security (20%)
    → VPC, security groups, WAF, Shield, Network Firewall
  Domain 4: Identity and Access Management (16%)
    → IAM policies, SCP, permission boundaries, federation
  Domain 5: Data Protection (18%)
    → KMS, S3 encryption, Secrets Manager, Certificate Manager
  Domain 6: Management and Security Governance (14%)
    → Organizations, Config, Control Tower, Security Hub

Study resources:
  - AWS Security documentation (free)
  - A Cloud Guru or Stephane Maarek Udemy course
  - AWS Security Workshops (workshops.aws)
  - Practise with ScoutSuite and Prowler against your own lab account
  - Jon Bonso practice exams (TutorialsDojo) — essential for exam-style prep
```

## Microsoft AZ-500 — Azure Security Engineer Associate

**The equivalent of AWS Security Specialty for Azure.**

```
Exam code:     AZ-500
Duration:      150 minutes
Questions:     ~40-60 (multiple choice, case studies, drag-and-drop)
Cost:          ~£165 GBP / $165 USD
Validity:      1 year (renewal required via free online assessment)
Prerequisites: Recommended: AZ-900 (Azure Fundamentals) first

Domain breakdown:
  Manage identity and access (25–30%)
    → Entra ID, RBAC, Conditional Access, PIM, managed identities
  Secure networking (20–25%)
    → NSGs, Azure Firewall, DDoS protection, Private Link
  Secure compute, storage, and databases (20–25%)
    → VM security, storage encryption, SQL security
  Manage security operations (25–30%)
    → Microsoft Defender for Cloud, Sentinel, Key Vault, monitoring

Study resources:
  - Microsoft Learn (free official learning paths for AZ-500)
  - John Savill's Azure Master Class (YouTube — free, excellent)
  - AZ-500 Study Guide by Microsoft Press
  - MeasureUp practice exams
  - Hands-on: ROADtools, ScoutSuite Azure, Prowler Azure against your own tenant
```

## Google Professional Cloud Security Engineer

```
Duration:      2 hours
Questions:     50-60 (multiple choice)
Cost:          ~$200 USD
Validity:      2 years

Domain breakdown:
  Configuring access within a cloud solution environment (27%)
    → GCP IAM, org policies, service accounts, workload identity
  Configuring network security (23%)
    → VPC, firewall rules, Private Google Access, VPC Service Controls
  Ensuring data protection (20%)
    → Cloud KMS, DLP, data classification, audit logs
  Managing operations within a cloud solution environment (22%)
    → Security Command Center, Cloud Audit Logs, binary authorization
  Supporting compliance requirements (8%)
    → Compliance frameworks, audit, regulatory requirements

Study resources:
  - Google Cloud Skills Boost (official learning paths)
  - Google Cloud Security Documentation
  - Professional Cloud Security Engineer Study Guide
  - Hands-on: GCPGoat, ScoutSuite GCP, Prowler GCP
```

## CCSP — Certified Cloud Security Professional (ISC2)

**Vendor-neutral, management-level cloud security cert. Valued in enterprise and consulting roles.**

```
Duration:      3 hours
Questions:     125 (multiple choice)
Cost:          $599 USD
Validity:      3 years (CPE required)
Prerequisites: 5 years IT experience including 3 in information security
               + 1 year in one of 6 cloud security domains
               (Or: CISSP holder = automatic CCSP waiver of experience)

Domain breakdown (equal weight ~17% each):
  Domain 1: Cloud Concepts, Architecture, and Design
  Domain 2: Cloud Data Security
  Domain 3: Cloud Platform and Infrastructure Security
  Domain 4: Cloud Application Security
  Domain 5: Cloud Security Operations
  Domain 6: Legal, Risk, and Compliance

Study resources:
  - (ISC)² Official Study Guide + CBK
  - Sybex CCSP Study Guide (Mike Chapple, David Seidl)
  - Certmike.com CCSP course
  - Best suited for: architects, consultants, compliance-focused roles
  - NOT an offensive/technical cert — theory and governance focused
```

## Prerequisite Path for Atharva's Context

Given your background (MSc Applied Cyber Security, CEH V12, AWS exposure via Eurostop work):

```
RECOMMENDED SEQUENCE:

Phase 1 — AWS Foundation (Month 1-2)
  → AWS Solutions Architect Associate (SAA-C03)
  → Validates cloud fundamentals for employer credibility
  → ~£150, passes in 2-3 months of study
  → Unlocks: you can credibly apply for cloud security roles

Phase 2 — Primary Target (Month 3-6)
  → AWS Certified Security — Specialty (SCS-C02)
  → Most impactful cert for cloud security job market
  → Combines with CEH + MSc = strong profile for Junior Cloud Security Analyst / Cloud Pentester
  → Study using this CLOUDHAWK curriculum + AWS documentation

Phase 3 — Azure Coverage (Month 6-9)
  → AZ-500
  → Most enterprises run hybrid AWS + Azure
  → Immediately validates your ROADtools and Azure AD work
  → Strong for consulting and MSP roles

Phase 4 — Practitioner Validation (Month 9-12)
  → CCSP or CompTIA Cloud+ (vendor neutral, enterprise recognised)
  → CCSP positions you for senior/architect roles and consulting
  → Or: focus on HTB/PNPT for offensive-focused roles

NOT RECOMMENDED AT THIS STAGE:
  → OSCP (valuable but not cloud-specific — do after AWS/Azure certs)
  → Google cert (lower market demand in UK/Singapore than AWS/Azure)
  → AWS Solutions Architect Professional (overkill before Security Specialty)
```

## Exam Prep Strategy

```
For any cloud security exam:

1. READ THE OFFICIAL DOCS FIRST (free)
   AWS: docs.aws.amazon.com/security
   Azure: learn.microsoft.com/security
   GCP: cloud.google.com/security

2. USE THIS CURRICULUM
   Every module in CLOUDHAWK covers exam-relevant material
   IAM, logging, encryption, network security = all exam domains

3. HANDS-ON LABS (30% of study time)
   CloudGoat for AWS Security Specialty
   AzureGoat for AZ-500
   GCPGoat for Google cert
   Doing > reading

4. PRACTICE EXAMS (final 2 weeks)
   Jon Bonso / TutorialsDojo (AWS) — 6 sets, score 80%+ before sitting
   MeasureUp (Azure) — official Microsoft practice platform
   Whizlabs (all platforms)

5. TIMING
   Study for 2-3 hours/day
   AWS SAA: 6-8 weeks
   AWS Security Specialty: 8-12 weeks
   AZ-500: 6-10 weeks
```
