# 10.3 — Community Resources and Further Reading

## What This Note Covers

The best blogs, newsletters, research papers, YouTube channels, communities, and whitepapers for staying current in cloud security — curated for practitioners who want depth, not surface-level summaries.

---

## Blogs and Research Sites

### Rhino Security Labs
```
URL: https://rhinosecuritylabs.com/blog/
Why: Publishes the most in-depth AWS attack research available.
     Original discoverers of most documented IAM privilege escalation paths.
     CloudGoat creators.
Must-reads:
  - "AWS IAM Privilege Escalation — Methods and Mitigation"
  - "Exploiting AWS Lambda (SSRF, privilege escalation)"
  - "AWS CloudTrail Blind Spots"
```

### Wiz Research
```
URL: https://www.wiz.io/blog/category/research
Why: Wiz discovers critical cloud vulnerabilities across all providers.
     "ChaosDB" (Cosmos DB breach), "OMIGOD" (Azure VM agent RCE),
     "ExtraReplica" (Azure PostgreSQL cross-tenant).
Must-reads:
  - "The Cloud Vulnerability Lifecycle" series
  - "Unmasking cloud misconfigurations" reports
  - Annual State of Cloud Security report
```

### Datadog Security Research
```
URL: https://securitylabs.datadoghq.com/
Why: Real-world cloud threat intelligence and attack analysis.
     Regular write-ups on actual AWS/Azure/GCP attack campaigns.
Must-reads:
  - "10 real-world stories of how we've compromised CI/CD pipelines"
  - Cloud attack surface reports
```

### Toni de la Fuente (Prowler Author)
```
URL: https://toniblyx.wordpress.com/
Twitter/X: @toniblyx
Why: Creator of Prowler. Deep AWS security expertise.
     Publishes practical AWS security guidance and benchmark updates.
```

### Scott Piper (flaws.cloud Author)
```
URL: https://summitroute.com/blog/
Why: One of the most respected AWS security researchers.
     Excellent IAM analysis, CloudTrail blind spots, misconfig research.
Must-reads:
  - "AWS IAM privilege escalation — a comprehensive list"
  - "Cloud security breaches" compilation
```

### Dirk-jan Mollema (ROADtools Author)
```
URL: https://dirkjanm.io/
Why: Creator of ROADtools and ROADtx. Deep Azure AD / Entra ID expertise.
     Original research on PRT attacks, device code phishing, Azure internals.
Must-reads:
  - "Abusing Azure AD SSO with the Primary Refresh Token"
  - "ROADtools: A framework to interact with Azure AD"
```

### Nick Frichette
```
URL: https://hackingthe.cloud/
Why: The single best reference for AWS attack techniques.
     Comprehensive documentation of every cloud attack technique
     with exact commands and real-world context.
     This is the companion reference to this curriculum.
```

```
URL: youtube.com/c/fwdcloudsec
```

## Key Whitepapers and Frameworks

### AWS
```
AWS Security Best Practices:
  https://docs.aws.amazon.com/wellarchitected/latest/security-pillar/

AWS Well-Architected Framework — Security Pillar:
  The official framework for secure AWS design.
  Covers IAM, detection, infrastructure protection, data protection.

AWS Security Reference Architecture:
  https://docs.aws.amazon.com/prescriptive-guidance/latest/security-reference-architecture/
  Real-world multi-account security architecture.

AWS CloudTrail Lake for Security Analysis:
  How to query CloudTrail at scale using Athena.
```

### Azure
```
Microsoft Cloud Security Benchmark:
  https://learn.microsoft.com/en-us/security/benchmark/azure/
  The definitive Azure security configuration reference.

Azure Security Documentation:
  https://learn.microsoft.com/en-us/azure/security/

Microsoft Defender for Cloud Best Practices:
  Covers all Defender plans and configuration.

Azure AD security operations guide:
  https://learn.microsoft.com/en-us/azure/active-directory/fundamentals/security-operations-introduction
```

### GCP
```
GCP Security Foundations Blueprint:
  https://cloud.google.com/architecture/security-foundations
  Google's recommended security architecture for enterprise GCP.

GCP Security Best Practices Centre:
  https://cloud.google.com/security/best-practices

Google Cloud Security Whitepapers:
  encryption, data access transparency, infrastructure security.
```

### MITRE ATT&CK for Cloud
```
URL: https://attack.mitre.org/matrices/enterprise/cloud/
Why: The definitive taxonomy for cloud attack techniques.
     Use for: mapping findings in reports, building detections,
     understanding what you're testing against.
```

## Staying Current — Workflow

```
DAILY (5 minutes):
  → Check CloudSecList or tl;dr sec newsletter
  → Twitter/X: follow @toniblyx, @_dirkjan, @ramimac, @scottpiper, @nfrichette

WEEKLY (30 minutes):
  → Read one new Rhino Security Labs or Wiz blog post
  → Check GitHub releases for tools you use (Pacu, ROADtools, ScoutSuite)

MONTHLY (2 hours):
  → Watch one fwd:cloudsec talk
  → Try one new CloudGoat scenario or HTB machine
  → Review MITRE ATT&CK for Cloud updates

QUARTERLY:
  → AWS/Azure/GCP re:Invent / Ignite / Next conference recordings
  → Review and update assessment checklists for new services
  → Check for new CloudTrail event names and GuardDuty finding types
```
