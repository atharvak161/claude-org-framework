# 05.6 — Serverless Reconnaissance and Assessment Methodology

## What This Note Covers

How to systematically assess serverless environments — discovery, enumeration, configuration review, and the specific reconnaissance techniques for event-driven architectures. Includes cold start timing attacks and the full serverless assessment checklist.

## Serverless Discovery — Finding the Functions

### AWS Lambda — Multi-Region Discovery

Lambda functions exist per-region. You must enumerate every region.

```bash
# Get all enabled regions
REGIONS=$(aws ec2 describe-regions \
  --query 'Regions[].RegionName' --output text)

# Enumerate Lambda functions in every region
for REGION in $REGIONS; do
  echo "=== $REGION ==="
  aws lambda list-functions \
    --region $REGION \
    --query 'Functions[].{Name:FunctionName,Runtime:Runtime,Role:Role,Modified:LastModified}' \
    --output table 2>/dev/null
done

# Also find functions via CloudFormation stacks
aws cloudformation list-stacks \
  --stack-status-filter CREATE_COMPLETE UPDATE_COMPLETE \
  --query 'StackSummaries[].StackName'

aws cloudformation describe-stack-resource \
  --stack-name STACK_NAME \
  --logical-resource-id LambdaFunction
```

### Finding Publicly Accessible Lambda URLs

AWS Lambda Function URLs are direct HTTPS endpoints without API Gateway.

```bash
# Check if a function has a Function URL (public endpoint)
aws lambda list-function-url-configs \
  --function-name FUNCTION_NAME \
  --region REGION

# If AuthType is NONE → publicly invocable without authentication
# FunctionUrl → the HTTPS endpoint

# Find functions with public URLs across all functions
for REGION in $(aws ec2 describe-regions --query 'Regions[].RegionName' --output text); do
  for FN in $(aws lambda list-functions --region $REGION \
    --query 'Functions[].FunctionName' --output text 2>/dev/null | tr '\t' '\n'); do
    URL_CONFIG=$(aws lambda list-function-url-configs \
      --function-name $FN --region $REGION 2>/dev/null)
    AUTH=$(echo $URL_CONFIG | python3 -c "import sys,json; \
      d=json.load(sys.stdin); \
      [print(u['AuthType'], u['FunctionUrl']) for u in d.get('FunctionUrlConfigs',[])]" 2>/dev/null)
    [ -n "$AUTH" ] && echo "$REGION/$FN: $AUTH"
  done
done
```

### API Gateway Endpoint Discovery

```bash
# List all REST APIs
aws apigateway get-rest-apis --region REGION \
  --query 'items[].{ID:id,Name:name,Created:createdDate}'

# Get deployed stages (active URLs)
aws apigateway get-stages \
  --rest-api-id API_ID \
  --region REGION \
  --query 'item[].{Stage:stageName,URL:join(``,["https://",`API_ID`,".execute-api.",`REGION`,".amazonaws.com/",stageName])}'

# List HTTP APIs (API Gateway v2)
aws apigatewayv2 get-apis --region REGION \
  --query 'Items[].{Name:Name,Endpoint:ApiEndpoint,Protocol:ProtocolType}'
```

## Serverless Configuration Deep Dive

### Full Lambda Function Audit

```bash
FUNCTION="target-function"
REGION="us-east-1"

# 1. Basic configuration
aws lambda get-function-configuration \
  --function-name $FUNCTION --region $REGION

# 2. Environment variables (secrets)
aws lambda get-function-configuration \
  --function-name $FUNCTION --region $REGION \
  --query 'Environment.Variables'

# 3. Execution role
ROLE_ARN=$(aws lambda get-function-configuration \
  --function-name $FUNCTION --region $REGION \
  --query 'Role' --output text)
ROLE_NAME=$(echo $ROLE_ARN | awk -F/ '{print $NF}')

aws iam list-attached-role-policies --role-name $ROLE_NAME
aws iam list-role-policies --role-name $ROLE_NAME

# 4. Download and inspect source code
CODE_URL=$(aws lambda get-function \
  --function-name $FUNCTION --region $REGION \
  --query 'Code.Location' --output text)
curl -o /tmp/function.zip "$CODE_URL"
unzip /tmp/function.zip -d /tmp/function-code/
grep -r "password\|secret\|key\|token\|os\.environ" /tmp/function-code/

# 5. Event source mappings (what triggers this function)
aws lambda list-event-source-mappings \
  --function-name $FUNCTION --region $REGION

# 6. Function policy (resource policy — who can invoke it)
aws lambda get-policy \
  --function-name $FUNCTION --region $REGION \
  --query 'Policy' | python3 -c "import sys,json; print(json.dumps(json.loads(json.load(sys.stdin)), indent=2))"

# 7. Aliases and versions
aws lambda list-aliases --function-name $FUNCTION --region $REGION
aws lambda list-versions-by-function --function-name $FUNCTION --region $REGION

# 8. Tags (may reveal environment/owner info)
aws lambda list-tags \
  --resource $ROLE_ARN --region $REGION
```

### Azure Function App Full Audit

```bash
APP="target-function-app"
RG="target-resource-group"

# 1. Function app details
az functionapp show --name $APP --resource-group $RG

# 2. All app settings (environment variables)
az functionapp config appsettings list --name $APP --resource-group $RG

# 3. Connection strings
az functionapp config connection-string list --name $APP --resource-group $RG

# 4. Managed identity
az functionapp show --name $APP --resource-group $RG --query "identity"

# 5. List individual functions
az functionapp function list --name $APP --resource-group $RG --output table

# 6. Check CORS settings
az functionapp cors show --name $APP --resource-group $RG

# 7. Authentication settings
az webapp auth show --name $APP --resource-group $RG
# "enabled": false → no authentication required on HTTP endpoints

# 8. Network access restrictions
az functionapp config access-restriction show --name $APP --resource-group $RG
```

## Cold Start Timing Attacks

A serverless function "cold starts" when it's invoked after being idle — the runtime environment is initialised from scratch. This takes longer (100ms-10s depending on runtime/size).

A warm function responds in milliseconds.

### Information Leakage via Timing

```bash
# Measure response time to detect warm vs cold
time curl -s "https://FUNCTION_URL" > /dev/null

# Repeated requests after a delay:
# First request after long idle → cold start (slow)
# Subsequent requests → warm (fast)

# Timing attack applications:
# 1. Rate limiting bypass detection
#    - If rate limiting is implemented in-memory → cold start resets the counter
#    - Send 100 requests → rate limited
#    - Wait for function to go cold (10-15 min idle)
#    - Send request → cold start → new in-memory state → rate limit reset

# 2. OTP/token validation timing
#    - If OTP check leaks timing (comparing strings with early exit):
#    - Measure response time for correct vs incorrect first character
#    - Build the OTP character by character via timing

# 3. Account enumeration via timing
#    - "User exists" path → DB lookup (slower)
#    - "User doesn't exist" path → immediate return (faster)
```

## Serverless Assessment Checklist

```
DISCOVERY:
[ ] Lambda functions enumerated across ALL regions
[ ] Function URLs found and tested for unauthenticated access
[ ] API Gateway APIs enumerated (REST + HTTP APIs)
[ ] Azure Function Apps listed in all resource groups
[ ] GCP Cloud Functions listed across all regions
[ ] Event source mappings documented

CONFIGURATION REVIEW (per function):
[ ] Environment variables reviewed for secrets
[ ] Execution role IAM permissions mapped
[ ] Runtime version checked (EOL = unpatched CVEs)
[ ] Timeout and memory settings noted
[ ] VPC configuration (if VPC-connected → can reach internal resources)
[ ] Function code downloaded and reviewed

CODE REVIEW:
[ ] Input validation — all event sources validated
[ ] Dangerous sinks checked: subprocess, os.system, eval, exec
[ ] SQL queries parameterised
[ ] Template rendering safe
[ ] Deserialization of untrusted data
[ ] Dependency versions checked against CVE databases

EXECUTION ROLE:
[ ] Least privilege assessment — does role have only what it needs?
[ ] Wildcard policies (Action:* or Resource:*) flagged
[ ] iam:* permissions checked (escalation path)
[ ] Cross-account trust relationships reviewed

TRIGGER SECURITY:
[ ] API Gateway authentication type per endpoint (NONE = unauthenticated)
[ ] WAF attached and configured
[ ] Rate limiting / throttling configured
[ ] S3 trigger bucket — can untrusted parties write to it?
[ ] SQS trigger queue — who has SendMessage permission?

CI/CD:
[ ] Workflow files reviewed for injection points
[ ] OIDC trust policies checked for overly broad conditions
[ ] Secrets storage reviewed (not in code, properly scoped)
[ ] Terraform state files located and reviewed
```

## Automated Serverless Security Scanning

```bash
# AWS — use Prowler for serverless-specific checks
prowler aws --services lambda apigateway

# Specific Prowler serverless checks:
prowler aws --check lambda_function_no_secrets_in_variables
prowler aws --check lambda_function_url_cors_policy
prowler aws --check lambda_function_url_auth_type
prowler aws --check apigateway_rest_api_no_public_access_without_authorizer

# ScoutSuite covers Lambda, API Gateway, and more
python3 scout.py aws --services lambda apigateway

# Manual OIDC trust policy audit
for ROLE in $(aws iam list-roles --query 'Roles[].RoleName' --output text); do
  TRUST=$(aws iam get-role --role-name $ROLE \
    --query 'Role.AssumeRolePolicyDocument' 2>/dev/null)
  echo $TRUST | grep -q "token.actions.githubusercontent.com" && \
    echo "GitHub OIDC role: $ROLE" && echo $TRUST | python3 -m json.tool
done
```

## Quick Reference — Serverless Recon Commands

```bash
# Lambda discovery (all regions)
for R in $(aws ec2 describe-regions --query 'Regions[].RegionName' --output text); do
  aws lambda list-functions --region $R \
    --query 'Functions[].FunctionName' --output text 2>/dev/null | \
    tr '\t' '\n' | sed "s/^/$R: /"
done

# Lambda full audit
aws lambda get-function-configuration --function-name NAME --region REGION
aws lambda get-function --function-name NAME --region REGION --query Code.Location
aws lambda get-policy --function-name NAME --region REGION

# API Gateway
aws apigateway get-rest-apis --region REGION
aws apigatewayv2 get-apis --region REGION

# Azure Function Apps
az functionapp list --output table
az functionapp config appsettings list --name APP --resource-group RG
az webapp auth show --name APP --resource-group RG

# GCP Cloud Functions
gcloud functions list --project PROJECT_ID
gcloud functions describe NAME --region REGION --format=json
gcloud functions get-iam-policy NAME --region REGION
```
