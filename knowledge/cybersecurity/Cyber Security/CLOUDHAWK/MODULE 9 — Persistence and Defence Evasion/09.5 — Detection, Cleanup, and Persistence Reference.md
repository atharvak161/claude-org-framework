# 09.5 — Detection, Cleanup, and Persistence Reference

## What This Note Covers

What cloud persistence and evasion looks like from the defender's perspective, how to detect persistence techniques in client environments, and engagement cleanup procedures for pentests.

---

## Detection Reference — AWS

### GuardDuty Findings Relevant to Persistence

```
PersistentSoftwareLauncher
  → Lambda function modified to include persistence mechanism

UnauthorizedAccess:IAMUser/ConsoleLoginSuccess.B
  → Console login from unexpected IP/browser/country

Recon:IAMUser/UserPermissions
  → Excessive IAM API enumeration calls

PrivilegeEscalation:IAMUser/AdministrativePermissions
  → IAM permission changes indicating escalation

Persistence:IAMUser/UserPermissions
  → IAM user/role/policy created or modified outside expected patterns

UnauthorizedAccess:IAMUser/InstanceCredentialExfiltration.OutsideAWS
  → EC2 instance credentials used from non-AWS IP
```

### CloudTrail Events to Alert On

These CloudTrail event names are the key monitoring targets for persistence detection:

```
iam:CreateUser                    → New IAM user
iam:CreateAccessKey               → New access key (any user)
iam:AttachUserPolicy              → Policy attached to user
iam:AttachRolePolicy              → Policy attached to role (especially AdministratorAccess)
iam:PutUserPolicy                 → Inline policy added
iam:UpdateAssumeRolePolicy        → Role trust policy modified
iam:CreatePolicyVersion           → New policy version created
sts:AssumeRoleWithWebIdentity     → OIDC/federation assumption (especially cross-account)
cloudtrail:StopLogging            → CloudTrail disabled
cloudtrail:DeleteTrail            → Trail deleted
lambda:UpdateFunctionCode         → Lambda code modified
events:PutRule                    → New EventBridge rule created
events:PutTargets                 → Target added to rule
secretsmanager:CreateSecret       → New secret created
ec2:ModifySnapshotAttribute       → Snapshot sharing modified
rds:StartExportTask               → Database export started
```

### Athena Queries for CloudTrail Analysis

```sql
-- Find all IAM user creation events
SELECT eventtime, useridentity.arn, requestparameters
FROM cloudtrail_logs
WHERE eventname = 'CreateUser'
  AND errorcode IS NULL
ORDER BY eventtime DESC;

-- Find all access key creation events
SELECT eventtime, useridentity.arn, requestparameters
FROM cloudtrail_logs
WHERE eventname = 'CreateAccessKey'
  AND errorcode IS NULL
ORDER BY eventtime DESC;

-- Find cross-account role assumptions
SELECT eventtime, useridentity.arn, requestparameters.roleArn
FROM cloudtrail_logs
WHERE eventname = 'AssumeRole'
  AND useridentity.accountid != requestparameters.roleArn_accountid
ORDER BY eventtime DESC;

-- Find Lambda function updates
SELECT eventtime, useridentity.arn, requestparameters.functionName
FROM cloudtrail_logs
WHERE eventname = 'UpdateFunctionCode20150331v2'
ORDER BY eventtime DESC;
```

## Detection Reference — Azure

### Azure Monitor Alerts for Persistence

```
Azure AD Audit: Add application credentials
  → App registration credential added

Azure AD Audit: Add owner to application
  → New owner added to app registration

Azure AD Audit: Update conditional access policy
  → CA policy modified (exclusions added)

Azure Activity: Create or update role assignment
  → RBAC role granted (especially Owner at subscription level)

Azure AD Sign-in: Sign-in from unfamiliar location
  → Auth from unexpected IP/country

Key Vault: SecretGet (after hours / unusual identity)
  → Secret access outside normal patterns
```

### KQL Queries for Azure Sentinel

```kql
// Find new app credentials added
AuditLogs
| where OperationName == "Add application credentials"
| project TimeGenerated, InitiatedBy, TargetResources
| order by TimeGenerated desc

// Find new Conditional Access exclusions
AuditLogs
| where OperationName == "Update conditional access policy"
| project TimeGenerated, InitiatedBy, TargetResources, AdditionalDetails
| order by TimeGenerated desc

// Find new Owner role assignments at subscription scope
AzureActivity
| where OperationNameValue == "MICROSOFT.AUTHORIZATION/ROLEASSIGNMENTS/WRITE"
| where Properties has "Owner"
| project TimeGenerated, Caller, ResourceGroup, Properties
| order by TimeGenerated desc
```

## Detection Reference — GCP

### Cloud Audit Log Queries (Cloud Logging)

```
resource.type="audited_resource"
protoPayload.methodName="google.iam.admin.v1.IAMPolicy.SetIamPolicy"
  → IAM policy modified at project/org level

protoPayload.methodName="google.iam.admin.v1.IAMService.CreateServiceAccount"
  → New service account created

protoPayload.methodName="google.iam.credentials.v1.IAMCredentials.GenerateAccessToken"
  → SA impersonation token generated

protoPayload.methodName="google.iam.admin.v1.IAMService.CreateServiceAccountKey"
  → New SA key created (persistence mechanism)

protoPayload.methodName="google.cloud.functions.v1.CloudFunctionsService.CreateFunction"
  → New Cloud Function deployed
```

```bash
# gcloud logging query for SA key creation
gcloud logging read \
  'protoPayload.methodName="google.iam.admin.v1.IAMService.CreateServiceAccountKey"' \
  --project PROJECT_ID \
  --freshness=7d \
  --format=json | python3 -m json.tool | grep -E "email|callerIp|time"
```

## Pentest Cleanup Procedures

After a cloud penetration test, all test artifacts must be removed. This is both professional obligation and critical — leaving backdoors in client environments is dangerous.

### AWS Cleanup Checklist

```bash
# Remove any IAM users created during testing
aws iam detach-user-policy \
  --user-name TEST_USER \
  --policy-arn arn:aws:iam::aws:policy/AdministratorAccess
aws iam delete-access-key \
  --user-name TEST_USER \
  --access-key-id AKIA...
aws iam delete-user --user-name TEST_USER

# Remove any access keys created on existing users
aws iam delete-access-key \
  --user-name EXISTING_USER \
  --access-key-id TEST_KEY_ID

# Revert any trust policy modifications
aws iam update-assume-role-policy \
  --role-name MODIFIED_ROLE \
  --policy-document file://original-trust-policy.json

# Remove any Lambda functions created
aws lambda delete-function --function-name TEST_FUNCTION

# Remove EventBridge rules created
aws events remove-targets --rule TEST_RULE --ids test-target
aws events delete-rule --name TEST_RULE

# Remove any snapshots created and shared
aws ec2 delete-snapshot --snapshot-id snap-TEST

# Remove any secrets created
aws secretsmanager delete-secret \
  --secret-id TEST_SECRET \
  --force-delete-without-recovery

# Remove any S3 objects written during testing
aws s3 rm s3://BUCKET/pentest-test-file.txt

# Verify cleanup
aws iam list-users | grep -i "pentest\|test\|svc-monitoring"
aws lambda list-functions | grep -i "test\|health-check\|beacon"
```

### Azure Cleanup Checklist

```bash
# Remove app registration credentials added
az ad app credential delete \
  --id APP_ID \
  --key-id CREDENTIAL_KEY_ID

# Remove any app owners added
az ad app owner remove \
  --id APP_ID \
  --owner-object-id YOUR_OBJECT_ID

# Revert Conditional Access policy changes
az rest --method PATCH \
  --url "https://graph.microsoft.com/v1.0/identity/conditionalAccess/policies/POLICY_ID" \
  --body '{"conditions":{"users":{"excludeUsers":[]}}}'

# Remove any role assignments created
az role assignment delete \
  --assignee TEST_SP_ID \
  --role Contributor \
  --scope /subscriptions/SUB_ID

# Remove any Function Apps created
az functionapp delete --name TEST_APP --resource-group RG
```

### GCP Cleanup Checklist

```bash
# Delete SA keys created
gcloud iam service-accounts keys delete KEY_ID \
  --iam-account SA_EMAIL \
  --project PROJECT_ID

# Delete service accounts created
gcloud iam service-accounts delete TEST_SA@PROJECT.iam.gserviceaccount.com \
  --project PROJECT_ID

# Remove IAM bindings added
gcloud projects remove-iam-policy-binding PROJECT_ID \
  --member="serviceAccount:TEST_SA@PROJECT.iam.gserviceaccount.com" \
  --role="roles/editor"

# Delete Cloud Functions created
gcloud functions delete TEST_FUNCTION --region REGION --project PROJECT_ID

# Delete Cloud Scheduler jobs created
gcloud scheduler jobs delete TEST_JOB --location REGION --project PROJECT_ID

# Verify cleanup
gcloud iam service-accounts list --project PROJECT_ID | grep -i test
gcloud functions list --project PROJECT_ID | grep -i test
```

## Persistence Technique Summary Table

| Technique | Platform | Persistence Type | Detection Difficulty | Requires |
|---|---|---|---|---|
| IAM backdoor user + key | AWS | High — permanent key | Medium | iam:CreateUser |
| Secondary access key | AWS | High — permanent key | Medium | iam:CreateAccessKey |
| Trust policy modification | AWS | Medium — role assumption | Low | iam:UpdateAssumeRolePolicy |
| Lambda scheduled beacon | AWS | Medium — survives credential rotation | Low | lambda:CreateFunction |
| App registration credential | Azure | High — permanent SP cred | Medium | Application Admin |
| CA policy exclusion | Azure | High — auth bypass | Low | CA Admin |
| Federated credential | Azure | High — OIDC trust | Very Low | Application Admin |
| SA JSON key | GCP | High — permanent key | Medium | iam.serviceAccountKeys.create |
| New SA with org role | GCP | High — survives project changes | Low | SA create + IAM modify |
| Org policy removal | GCP | Medium — unlocks further techniques | Low | orgpolicy.policies.create |
| Cloud Function beacon | AWS/GCP | Medium | Low | function create + schedule |

## Quick Reference — Detection and Cleanup

```bash
# AWS: Audit IAM for test artifacts
aws iam list-users
aws iam list-roles | grep -i "test\|pentest\|attacker"
aws lambda list-functions --query 'Functions[].FunctionName'

# AWS: Cleanup
aws iam delete-user --user-name TEST_USER
aws lambda delete-function --function-name TEST_FUNCTION
aws events delete-rule --name TEST_RULE

# Azure: Audit
az ad app list --all | grep -i test
az role assignment list --all | grep -i test

# Azure: Cleanup
az ad app credential delete --id APP_ID --key-id KEY_ID
az role assignment delete --assignee SP_ID --role Contributor --scope /subscriptions/SUB

# GCP: Audit
gcloud iam service-accounts list --project PROJECT_ID
gcloud functions list --project PROJECT_ID

# GCP: Cleanup
gcloud iam service-accounts keys delete KEY_ID --iam-account SA_EMAIL
gcloud iam service-accounts delete SA_EMAIL
gcloud functions delete FUNCTION --region REGION
```
