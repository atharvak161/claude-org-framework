## gcloud CLI — Security Assessment Usage

The official GCP CLI is the primary tool for every GCP engagement. Beyond basic enumeration, it has several security-assessment-specific uses.

```bash
# Install (if not already installed via GCP SDK)
curl https://sdk.cloud.google.com | bash
exec -l $SHELL
gcloud init

# Auth patterns for engagements
gcloud auth login                              # interactive browser login
gcloud auth activate-service-account --key-file key.json  # SA key auth
gcloud auth print-access-token                 # get current token
gcloud config set project PROJECT_ID           # set working project

# Systematic IAM enumeration across all projects
for PROJECT in $(gcloud projects list --format="value(projectId)"); do
  echo "=== $PROJECT ==="
  gcloud projects get-iam-policy $PROJECT 2>/dev/null \
    --flatten="bindings[].members" \
    --format="table(bindings.role, bindings.members)"
done

# Find all service accounts and their keys across projects
for PROJECT in $(gcloud projects list --format="value(projectId)"); do
  echo "=== $PROJECT ==="
  for SA in $(gcloud iam service-accounts list --project $PROJECT \
    --format="value(email)" 2>/dev/null); do
    KEY_COUNT=$(gcloud iam service-accounts keys list \
      --iam-account $SA --project $PROJECT \
      --managed-by user \
      --format="value(name)" 2>/dev/null | wc -l)
    if [ "$KEY_COUNT" -gt 0 ]; then
      echo "SA with user-managed keys: $SA (keys: $KEY_COUNT)"
    fi
  done
done

# Find all publicly accessible Cloud Functions
for PROJECT in $(gcloud projects list --format="value(projectId)"); do
  for REGION in us-central1 us-east1 europe-west1 asia-east1; do
    gcloud functions list --project $PROJECT --regions $REGION \
      --format="table(name, httpsTrigger.url, status)" 2>/dev/null
  done
done
```

## ScoutSuite — GCP Misconfiguration Scanning

ScoutSuite supports GCP alongside AWS and Azure. It performs automated misconfiguration scanning across all GCP services.

```bash
# Install (from source)
git clone https://github.com/nccgroup/ScoutSuite
cd ScoutSuite
pip3 install virtualenv
virtualenv -p python3 venv
source venv/bin/activate
pip install -r requirements.txt

# Run against GCP (user account auth)
python3 scout.py gcp --user-account --project PROJECT_ID

# Run with service account key file
python3 scout.py gcp \
  --service-account key.json \
  --project PROJECT_ID

# Run across all projects in an org
python3 scout.py gcp \
  --user-account \
  --all-projects

# Output: scoutsuite-report/
# Open: scoutsuite-report/scoutsuite-results/scoutsuite_results_gcp-*.js
```

**What ScoutSuite checks for GCP:**
- IAM: primitive roles (`roles/owner`, `roles/editor`) used directly, external users with privileged roles
- GCS: public buckets, buckets without uniform access control, buckets without logging
- Compute: instances with public IPs, no OS Login, serial port access enabled, default SA attached
- Cloud SQL: public IPs, no SSL required, no backups
- GKE: legacy ABAC enabled, no network policy, dashboard enabled
- Cloud Functions: unauthenticated invocation, overprivileged execution SA
- Logging: audit logs disabled, log exports not configured

## GCPBucketBrute — Bucket Enumeration

```bash
# Install
git clone https://github.com/RhinoSecurityLabs/GCPBucketBrute
cd GCPBucketBrute
pip3 install -r requirements.txt

# Unauthenticated check — tests if bucket exists and is public
python3 gcpbucketbrute.py -k targetcompany -u

# Authenticated check — also tests if you have read access
python3 gcpbucketbrute.py -k targetcompany

# Use a custom keyword list
python3 gcpbucketbrute.py -k targetcompany -f keywords.txt

# Output: open buckets + their public permission level
# "ReadACL": can read ACLs
# "ListBucket": can list objects
# "ReadObject": can read objects
# "WriteObject": can write objects (critical!)
```

## GCP-IAM-Privilege-Escalation — Automated Privesc Scanning

```bash
# Install
git clone https://github.com/RhinoSecurityLabs/GCP-IAM-Privilege-Escalation
cd GCP-IAM-Privilege-Escalation
pip3 install -r requirements.txt

# Step 1: Enumerate all permissions for current identity
python3 PrivEscScanner/enumerate_member_permissions.py \
  --project-id PROJECT_ID

# Output: JSON file listing all permissions the current identity has

# Step 2: Check for privilege escalation paths
python3 PrivEscScanner/check_for_privesc.py \
  --project-id PROJECT_ID

# Output: list of exploitable privilege escalation techniques available
# with the current permission set

# Step 3: Run specific escalation method (if found)
# Methods are in PrivEscMethods/ directory
python3 PrivEscMethods/[TECHNIQUE].py --project-id PROJECT_ID
```

## Hayat — GCP Security Scanner

Hayat is a GCP-specific security scanner focused on IAM misconfigurations and dangerous permission combinations.

```bash
# Install
git clone https://github.com/DenizParlak/hayat
cd hayat
pip3 install -r requirements.txt

# Run
python3 hayat.py

# Hayat checks:
# - Service accounts with primitive roles
# - Users with direct primitive role assignments
# - External (outside-domain) users with access
# - SAs with user-managed keys (key sprawl)
# - Public GCS buckets
# - Unauthenticated Cloud Functions
```

## Forseti Security — Continuous GCP Compliance Monitoring

Forseti is a Google-developed open-source tool for continuous GCP security monitoring. More relevant for defenders but understanding it helps attackers understand what's being monitored.

```bash
# Forseti is deployed as a GCP deployment
# Relevant for understanding what a mature GCP security posture looks like:
# - Inventory scanning
# - Policy violation detection
# - IAM explain (shows why an identity has a given permission)

# As an attacker: check if Forseti is deployed
gcloud compute instances list --filter="name:forseti" --project PROJECT_ID
# If found: Forseti is actively monitoring — be aware
```

## Practice Resources — GCP Specific

### GCPGoat

```bash
# Intentionally vulnerable GCP environment by INE
git clone https://github.com/ine-labs/GCPGoat
cd GCPGoat

# Prerequisites: gcloud + Terraform
gcloud auth application-default login
terraform init
terraform apply  # deploys vulnerable GCP infrastructure

# Scenarios include:
# - Service account key exposure via public GCS bucket
# - Cloud Function with overprivileged SA
# - GCS bucket public access misconfiguration
# - IAM privilege escalation via custom role
# - Metadata service credential theft

# Destroy when done
terraform destroy
```
