# 07.1 — Unauthenticated Cloud Asset Discovery

## What This Note Covers

What an attacker can discover about a target's cloud infrastructure without any credentials — tenant IDs, account IDs, storage resources, IP ranges, exposed services, and the cloud services hidden in DNS records.

---

## Why Unauthenticated Recon Matters

Before any phishing, credential stuffing, or exploitation, a cloud attacker needs to map the attack surface. Cloud infrastructure leaks significant information publicly:

- AWS account IDs appear in S3 URLs, AMI owner fields, and error messages
- Azure tenant IDs are exposed via login federation endpoints
- GCP project IDs appear in GCS bucket URLs and metadata
- Storage buckets are globally nameable — enumerable by wordlist
- Cloud-hosted services appear in DNS records, CT logs, and Shodan
- IP ranges for all three major providers are publicly documented

Everything in this note requires zero credentials.

---

## AWS — Unauthenticated Discovery

### Account ID Discovery

```bash
# Method 1: Error messages from S3
# Attempt to access a non-existent bucket policy:
curl -v https://TARGET-BUCKET.s3.amazonaws.com/?policy 2>&1 | grep -i "access denied\|account"

# Method 2: Public AMI owner field
# AMIs owned by the target account are tagged with the account ID
aws ec2 describe-images \
  --owners self \
  --region us-east-1 \
  --no-sign-request 2>&1
# If any AMIs are public, --owners self won't work without auth
# But known AMI IDs from public sources reveal owner account IDs:
aws ec2 describe-images \
  --image-ids ami-0TARGET123 \
  --region us-east-1 \
  --no-sign-request \
  --query 'Images[].OwnerId'

# Method 3: SNS/SQS ARNs in application source code
# ARN format: arn:aws:sns:us-east-1:ACCOUNT_ID:TopicName
grep -r "arn:aws:" . | grep -oP "arn:aws:[^:]+:[^:]*:(\d{12})" | sort -u

# Method 4: Public ECR repositories (some are intentionally public)
aws ecr-public describe-repositories --no-sign-request --region us-east-1
```

### S3 Bucket Discovery (No Credentials)

```bash
# Tool: S3Scanner
pip3 install s3scanner
s3scanner scan --bucket-file wordlist.txt
s3scanner scan --bucket target-company-backups

# Manual — check if a bucket exists and its access level
aws s3 ls s3://TARGET-BUCKET --no-sign-request
# "AccessDenied" → bucket exists, private
# Object list → bucket exists, PUBLIC READ
# "NoSuchBucket" → doesn't exist

# Attempt to list the bucket via HTTPS
curl -s https://TARGET-BUCKET.s3.amazonaws.com/ | head -50
# Public bucket returns XML listing with object keys

# Generate bucket name wordlist from company name
python3 -c "
company = 'targetcompany'
suffixes = ['', '-dev', '-prod', '-staging', '-backup', '-backups', '-data', '-logs',
            '-assets', '-static', '-media', '-files', '-uploads', '-archive', '-storage',
            '-terraform', '-tf-state', '-tfstate', '-config', '-configs', '-infra',
            '-internal', '-private', '-public', '-web', '-api', '-app']
for s in suffixes:
    print(f'{company}{s}')
    print(f'{s[1:]}-{company}') if s else None
" > buckets.txt
s3scanner scan --bucket-file buckets.txt
```

### CloudFront and Exposed Services

```bash
# CloudFront distributions expose origin domains
# A CNAME of *.cloudfront.net in DNS reveals CloudFront usage
# The CloudFront distribution ID is in HTTP response headers:
curl -I https://TARGET-SITE.com | grep -i "x-amz\|cloudfront"
# X-Cache: Hit from cloudfront → behind CloudFront

# API Gateway endpoints are discoverable via DNS and port scanning
# Format: APIID.execute-api.REGION.amazonaws.com
# Look for these in JS files, mobile app traffic, API docs
```

## Azure — Unauthenticated Discovery

### Tenant ID Discovery

```bash
# Tenant ID is exposed via the OpenID Connect discovery endpoint
curl -s "https://login.microsoftonline.com/TARGET-DOMAIN.com/.well-known/openid-configuration" \
  | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['issuer'].split('/')[3])"

# AADInternals (PowerShell) — comprehensive tenant recon
Invoke-AADIntReconAsOutsider -DomainName "target.com"
# Returns: tenant name, tenant ID, federation info, domains, brand

# Alternative via user realm discovery
curl -s "https://login.microsoftonline.com/getuserrealm.srf?login=anyuser@TARGET-DOMAIN.com&xml=1"
# Returns: NameSpaceType (Managed/Federated), FederationBrandName, DomainName
```

### Azure Blob Storage Discovery

```bash
# Storage account naming convention: all lowercase, 3-24 chars, alphanumeric only
# URL: https://ACCOUNTNAME.blob.core.windows.net/CONTAINER/BLOB

# Common storage account name patterns
python3 -c "
company = 'targetcompany'
# Remove special chars, lowercase
base = ''.join(c for c in company.lower() if c.isalnum())
suffixes = ['', 'dev', 'prod', 'stg', 'backup', 'data', 'logs', 'static', 'assets', 'media',
            'files', 'archive', 'storage', 'config', 'tfstate', 'terraform']
for s in suffixes:
    print(f'{base}{s}')
    print(f'{s}{base}') if s else None
" > azure-accounts.txt

# Check each with cloud_enum
pip3 install cloud_enum
python3 cloud_enum.py -k targetcompany --disable-aws --disable-gcp

# Manual check
curl -s "https://ACCOUNTNAME.blob.core.windows.net/CONTAINER/?restype=container&comp=list"
# Public container: returns XML blob listing
# Private: 403 PublicAccessNotPermitted
```

### Azure App Services Discovery

```bash
# Azure App Service default domains: APP_NAME.azurewebsites.net
# Azure Functions: APP_NAME.azurewebsites.net or APP_NAME.azurefd.net
# Azure Static Web Apps: APP_NAME.azurestaticapps.net
# Azure API Management: APIM_NAME.azure-api.net

# Enumerate via DNS brute force and CT logs
# Tools: amass, subfinder, httpx — targeting azurewebsites.net subdomains
amass enum -d azurewebsites.net -passive
```

## GCP — Unauthenticated Discovery

### Project ID Discovery

```bash
# GCP project IDs appear in:
# - GCS bucket URLs: gs://BUCKET_NAME (bucket name often contains project)
# - Error messages from GCP APIs
# - Source code and config files
# - Cloud Run URLs: SERVICE-HASH-REGION.a.run.app
# - App Engine URLs: PROJECT_ID.appspot.com

# Check if an App Engine app exists for a project
curl -I https://TARGET_PROJECT_ID.appspot.com
# 200 OK → App Engine app exists and is reachable

# GCS bucket URL reveals project implicitly
curl -s https://storage.googleapis.com/TARGET-BUCKET/
```

### GCS Bucket Discovery

```bash
# Tool: GCPBucketBrute
git clone https://github.com/RhinoSecurityLabs/GCPBucketBrute
python3 gcpbucketbrute.py -k targetcompany -u  # unauthenticated

# GCS bucket URL formats:
# https://storage.googleapis.com/BUCKET_NAME/
# https://BUCKET_NAME.storage.googleapis.com/
# gs://BUCKET_NAME/

# Direct check
curl -s "https://storage.googleapis.com/TARGET-BUCKET/"
# Public: XML listing or object content
# Private: AccessDenied XML

# GCS public bucket via gsutil (no auth)
gsutil ls gs://TARGET-BUCKET
```

## IP Range Enumeration — Identifying Cloud Assets

All three major providers publish their IP ranges publicly.

```bash
# AWS IP ranges
curl -s https://ip-ranges.amazonaws.com/ip-ranges.json | python3 -c "
import sys, json
data = json.load(sys.stdin)
# Get all CloudFront IPs
cf_ips = [p['ip_prefix'] for p in data['prefixes'] if p['service'] == 'CLOUDFRONT']
print('\n'.join(cf_ips[:10]))
# Services: AMAZON, EC2, S3, CLOUDFRONT, ROUTE53, GLOBALACCELERATOR, etc.
"

# Azure IP ranges
# https://www.microsoft.com/en-us/download/details.aspx?id=56519
# Download and parse the JSON — updated weekly

# GCP IP ranges
curl -s https://www.gstatic.com/ipranges/cloud.json | python3 -c "
import sys, json
data = json.load(sys.stdin)
for p in data.get('prefixes', [])[:10]:
    print(p.get('ipv4Prefix', p.get('ipv6Prefix', '')), p.get('service',''), p.get('scope',''))
"

# Given a target IP — identify which cloud provider owns it
# Use ipinfo.io or WHOIS:
curl https://ipinfo.io/TARGET_IP/org
# Returns: "AS16509 Amazon.com, Inc." or "AS8075 Microsoft Corporation" etc.
```

## DNS Enumeration for Cloud Infrastructure

DNS records reveal which cloud services a target uses.

```bash
# CNAME records pointing to cloud services
dig TARGET-DOMAIN.com CNAME
nslookup TARGET-DOMAIN.com
# Common cloud CNAMEs to look for:
# *.s3.amazonaws.com           → S3 bucket
# *.cloudfront.net             → CloudFront CDN
# *.azurewebsites.net          → Azure App Service
# *.blob.core.windows.net      → Azure Blob Storage
# *.azurefd.net                → Azure Front Door
# *.a.run.app                  → GCP Cloud Run
# *.appspot.com                → GCP App Engine
# *.cloudfunctions.net         → GCP Cloud Functions
# *.elb.amazonaws.com          → AWS ELB
# *.execute-api.*.amazonaws.com → AWS API Gateway

# MX records — identify email hosting (SaaS indicator)
dig TARGET-DOMAIN.com MX
# google.com → Google Workspace
# protection.outlook.com → Microsoft 365
# amazonses.com → Amazon SES (AWS email)

# TXT records — SPF/DKIM reveal used services
dig TARGET-DOMAIN.com TXT
# include:amazonses.com → uses AWS SES
# include:_spf.google.com → uses Google Workspace
# include:spf.protection.outlook.com → uses Microsoft 365
```
