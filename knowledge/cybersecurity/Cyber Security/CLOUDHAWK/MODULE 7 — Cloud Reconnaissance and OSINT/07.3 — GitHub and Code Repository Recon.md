# 07.3 — GitHub and Code Repository Recon

## What This Note Covers

Finding cloud credentials and infrastructure intelligence in public code repositories — GitHub dorks, automated secret scanning with truffleHog and gitleaks, Docker Hub layer analysis, and organisation-wide repository enumeration.

---

## Why Code Repositories Are the Richest Target

Developers commit secrets constantly. Studies consistently find that thousands of valid cloud credentials are committed to public GitHub repositories every day. Common sources:

- AWS access keys hardcoded in scripts
- Service account JSON files committed alongside application code
- `.env` files accidentally pushed with credentials
- CI/CD configuration files with embedded secrets
- Terraform variable files with plaintext passwords
- Docker Compose files with database credentials
- Application config files with API keys

## Automated Secret Scanning — truffleHog

truffleHog scans git history and file content for secrets using entropy analysis and regex patterns. It also verifies whether detected secrets are still active.

```bash
# Install
pip3 install trufflehog

# Scan a GitHub repository (including full git history)
trufflehog github --repo https://github.com/TARGET_ORG/TARGET_REPO

# Scan all repos in an organisation
trufflehog github --org TARGET_ORG

# Scan a local repository
trufflehog git file:///path/to/local/repo

# Scan a local filesystem (not a git repo)
trufflehog filesystem /path/to/scan

# Scan Docker image layers
trufflehog docker --image IMAGE_NAME

# Scan S3 bucket contents
trufflehog s3 --bucket BUCKET_NAME

# Only show verified secrets (actively valid — highest confidence)
trufflehog github --repo REPO_URL --only-verified

# Output in JSON for parsing
trufflehog github --repo REPO_URL --json | python3 -m json.tool

# Key output fields:
# DetectorName: AWS, GCP, Azure, GitHub, Slack, etc.
# Verified: true/false (actively tested)
# Raw: the detected secret value
# SourceMetadata: file path, line number, commit hash
```

## Automated Secret Scanning — gitleaks

gitleaks is a fast, rules-based secret scanner for git repositories.

```bash
# Install
brew install gitleaks    # macOS
# Or download binary from: github.com/gitleaks/gitleaks/releases

# Scan a local repository
gitleaks detect --source /path/to/repo --report-path report.json

# Scan a specific commit range
gitleaks detect --source /path/to/repo \
  --log-opts="COMMIT_HASH1..COMMIT_HASH2"

# Scan staged files only (pre-commit hook use)
gitleaks protect --staged

# Scan with verbose output (shows each finding)
gitleaks detect --source . -v

# Custom rules (extend default ruleset)
gitleaks detect --source . --config custom-rules.toml

# Output formats: json, csv, sarif, junit
gitleaks detect --source . --report-format json --report-path findings.json
```

## Organisation-Wide GitHub Enumeration

When targeting a specific company, enumerate all their public GitHub repositories.

```bash
# Using GitHub CLI (gh)
gh auth login

# List all repos for an organisation
gh repo list TARGET_ORG --limit 1000 --json name,url,isPrivate,updatedAt \
  | python3 -c "
import sys, json
repos = json.load(sys.stdin)
public = [r for r in repos if not r['isPrivate']]
print(f'Public repos: {len(public)}')
for r in public:
    print(r['url'])
"

# Clone all public repos and scan
gh repo list TARGET_ORG --limit 1000 --json url \
  | python3 -c "import sys,json; [print(r['url']) for r in json.load(sys.stdin)]" \
  | while read url; do
    repo_name=$(echo $url | awk -F/ '{print $NF}')
    git clone --depth=1 $url /tmp/scan/$repo_name 2>/dev/null
    gitleaks detect --source /tmp/scan/$repo_name --report-path /tmp/scan/$repo_name-report.json -q
  done

# Aggregate findings
find /tmp/scan -name "*-report.json" | xargs python3 -c "
import sys, json
for f in sys.argv[1:]:
    try:
        findings = json.load(open(f))
        for finding in findings:
            print(f['File'], f.get('RuleID'), str(f.get('Secret',''))[:50])
    except: pass
"
```

## Git History Mining

Even if a secret was deleted in a later commit, it persists in git history. Full history scanning catches these.

```bash
# Clone with full history
git clone https://github.com/TARGET_ORG/TARGET_REPO
cd TARGET_REPO

# Search entire git history for specific patterns
git log --all --full-history --oneline | head -50

# Search all commits for AWS key pattern
git log -p --all | grep -P "AKIA[0-9A-Z]{16}"

# Search for any credential-like pattern in history
git log -p --all | grep -iE \
  "password|secret|api_key|private_key|access_key|service_account" | head -100

# Find deleted files that contained secrets
git log --all --full-history -- "**/.env" "**/*.pem" "**/credentials*"

# Show content of a file at a specific commit
git show COMMIT_HASH:path/to/file

# List all files ever committed (including deleted)
git log --all --full-history --name-only --format="" | sort -u | grep -iE "env|secret|credential|key|password"
```

## Docker Hub Layer Analysis

Public Docker images may contain secrets in their build layers.

```bash
# Search Docker Hub for target organisation images
curl -s "https://hub.docker.com/v2/repositories/TARGETORG/?page_size=100" \
  | python3 -c "
import sys, json
data = json.load(sys.stdin)
for r in data.get('results', []):
    print(r['name'], r['pull_count'])
"

# Pull and scan a specific image
docker pull TARGETORG/IMAGE_NAME:latest
trufflehog docker --image TARGETORG/IMAGE_NAME:latest

# Manual layer extraction
docker save TARGETORG/IMAGE_NAME -o image.tar
mkdir /tmp/image-layers && tar xf image.tar -C /tmp/image-layers/
for layer in /tmp/image-layers/*/layer.tar; do
  mkdir /tmp/extracted
  tar xf $layer -C /tmp/extracted 2>/dev/null
  grep -rP "AKIA[0-9A-Z]{16}" /tmp/extracted/ 2>/dev/null
  grep -r '"type": "service_account"' /tmp/extracted/ 2>/dev/null
  rm -rf /tmp/extracted
done
```
