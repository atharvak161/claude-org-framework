# 07.2 — Subdomain Enumeration and Certificate Transparency

## What This Note Covers

How to discover cloud-hosted assets via subdomain enumeration and certificate transparency logs — identifying S3 buckets, App Services, Cloud Run endpoints, and other cloud resources that are exposed through DNS.

---

## Why Subdomains Reveal Cloud Infrastructure

Organisations map custom domains to cloud services via CNAME records. A CNAME pointing to `*.s3.amazonaws.com` reveals an S3 bucket. One pointing to `*.azurewebsites.net` reveals an App Service. Enumerate all subdomains of a target domain and you get a map of their cloud footprint.

Additionally, certificate transparency (CT) logs record every TLS certificate issued for a domain — including certificates for internal subdomains the organisation may not expect to be public.

---

## Certificate Transparency — Passive Subdomain Discovery

Every TLS certificate is logged in public CT logs. This means every subdomain a target has ever used with HTTPS is discoverable without any active scanning.

```bash
# crt.sh — web UI and API
curl -s "https://crt.sh/?q=%.targetcompany.com&output=json" | \
  python3 -c "
import sys, json
certs = json.load(sys.stdin)
domains = set()
for c in certs:
    name = c.get('name_value', '')
    for d in name.split('\n'):
        d = d.strip().lstrip('*.')
        if d:
            domains.add(d)
for d in sorted(domains):
    print(d)
" | sort -u > subdomains-ct.txt

# Process crt.sh results for cloud service identification
grep -iE "s3\.|cloudfront\|azurewebsites\|blob\.core\|a\.run\.app\|appspot\|cloudfunctions" subdomains-ct.txt

# Alternative CT sources
# Censys: https://search.censys.io/certificates
# Facebook CT: https://developers.facebook.com/tools/ct/
# Google CT: https://transparencyreport.google.com/https/certificates
```

## Active Subdomain Enumeration

CT logs catch historical certificates. Active enumeration finds current and additional subdomains.

### amass — Comprehensive Subdomain Discovery

```bash
# Install
go install -v github.com/owasp-amass/amass/v4/...@master
# Or: apt install amass / brew install amass

# Passive enumeration (no direct DNS queries to target)
amass enum -passive -d targetcompany.com -o amass-passive.txt

# Active enumeration (DNS brute force + APIs)
amass enum -active -d targetcompany.com \
  -brute \
  -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-20000.txt \
  -o amass-active.txt

# DNS brute force only
amass enum -d targetcompany.com -brute -w wordlist.txt

# Track changes over time (run periodically)
amass track -d targetcompany.com
```

### subfinder — Fast Passive Enumeration

```bash
# Install
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest

# Basic enumeration (uses many OSINT sources)
subfinder -d targetcompany.com -o subfinder-results.txt

# With API keys configured (more results)
subfinder -d targetcompany.com -all -o subfinder-all.txt

# Enumerate multiple domains
subfinder -dL domains.txt -o subfinder-bulk.txt
```

### dnsx — DNS Resolution and Filtering

```bash
# Install
go install -v github.com/projectdiscovery/dnsx/cmd/dnsx@latest

# Resolve and filter live subdomains from discovered list
cat subdomains-ct.txt subfinder-results.txt amass-passive.txt | \
  sort -u | \
  dnsx -resp -o live-subdomains.txt

# Output includes: subdomain → IP address → DNS record type
```

## Identifying Cloud Services from Subdomains

Once you have a list of live subdomains, identify which cloud services they point to:

```bash
# Extract CNAMEs for all live subdomains
while read subdomain; do
  cname=$(dig +short CNAME $subdomain 2>/dev/null | head -1)
  [ -n "$cname" ] && echo "$subdomain → $cname"
done < live-subdomains.txt | tee cname-map.txt

# Filter by cloud provider
grep -i "amazonaws.com" cname-map.txt     # AWS
grep -i "azure\|windows.net\|msecnd.net" cname-map.txt  # Azure
grep -i "google\|appspot\|run.app\|cloudfunctions" cname-map.txt  # GCP
grep -i "cloudfront.net" cname-map.txt    # AWS CloudFront
grep -i "azurefd.net\|trafficmanager.net" cname-map.txt  # Azure Front Door

# HTTP probe all live subdomains for response details
cat live-subdomains.txt | \
  httpx -sc -title -server -td -o http-probe-results.txt

# Look for cloud-specific response headers
grep -i "x-amz\|x-ms-\|x-goog-\|cloudfront\|AmazonS3" http-probe-results.txt
```

## Subdomain Takeover — Cloud-Specific

A subdomain takeover occurs when a CNAME points to a cloud resource that no longer exists. The attacker creates that resource and hijacks the subdomain.

### Detection

```bash
# Tool: subjack — checks for takeover vulnerability
go install github.com/haccer/subjack@latest
subjack -w live-subdomains.txt -t 100 -timeout 30 -o takeovers.txt -ssl

# Tool: nuclei with takeover templates
nuclei -l live-subdomains.txt -t ~/nuclei-templates/takeovers/

# Manual check pattern:
# 1. Find CNAME: assets.targetcompany.com → targetcompany-assets.s3.amazonaws.com
# 2. Check if bucket exists: aws s3 ls s3://targetcompany-assets --no-sign-request
# 3. "NoSuchBucket" → takeover possible
# 4. Create the bucket in your AWS account → you now control the subdomain
```

### Cloud-Specific Takeover Indicators

```bash
# S3 bucket takeover
curl https://TARGET-SUBDOMAIN.targetcompany.com
# Response: "NoSuchBucket" → vulnerable

# Azure App Service takeover
curl https://TARGET-SUBDOMAIN.targetcompany.com
# Response: "404 Web Site not found" from azurewebsites.net → vulnerable

# GCP Cloud Run/App Engine takeover
curl https://TARGET-SUBDOMAIN.targetcompany.com
# Response: "Error: Page not found" from Google → check if project/service still exists
```

## Wildcard DNS and Cloud Discovery

Some organisations use wildcard DNS (`*.targetcompany.com`) pointing to cloud infrastructure. This is discoverable:

```bash
# Check for wildcard DNS
dig RANDOMSTRING.targetcompany.com
# If it resolves to a real IP → wildcard exists

# Identify what the wildcard points to
dig *.targetcompany.com
curl -H "Host: nonexistent.targetcompany.com" https://IP_OF_WILDCARD/
# Response tells you what CDN/service is handling it
```

## Automated Cloud Recon Pipeline

```bash
#!/bin/bash
TARGET="targetcompany.com"
OUTPUT_DIR="./recon-$TARGET"
mkdir -p $OUTPUT_DIR

# Phase 1: CT logs
echo "[*] Certificate Transparency..."
curl -s "https://crt.sh/?q=%.$TARGET&output=json" | \
  python3 -c "import sys,json; [print(c['name_value']) for c in json.load(sys.stdin)]" | \
  tr ',' '\n' | sed 's/\*\.//' | sort -u > $OUTPUT_DIR/ct-subdomains.txt

# Phase 2: Passive enumeration
echo "[*] Subfinder..."
subfinder -d $TARGET -o $OUTPUT_DIR/subfinder.txt -silent

# Phase 3: Combine and resolve
cat $OUTPUT_DIR/ct-subdomains.txt $OUTPUT_DIR/subfinder.txt | \
  sort -u | \
  dnsx -silent -o $OUTPUT_DIR/live-subdomains.txt

# Phase 4: HTTP probe
echo "[*] HTTP probing..."
httpx -l $OUTPUT_DIR/live-subdomains.txt \
  -sc -title -server -td \
  -o $OUTPUT_DIR/http-results.txt -silent

# Phase 5: CNAME mapping for cloud identification
echo "[*] CNAME mapping..."
while read sub; do
  cname=$(dig +short CNAME $sub 2>/dev/null | head -1)
  [ -n "$cname" ] && echo "$sub → $cname"
done < $OUTPUT_DIR/live-subdomains.txt > $OUTPUT_DIR/cname-map.txt

# Phase 6: Takeover check
echo "[*] Takeover check..."
subjack -w $OUTPUT_DIR/live-subdomains.txt \
  -t 100 -timeout 30 \
  -o $OUTPUT_DIR/takeovers.txt -ssl

echo "[*] Done. Results in $OUTPUT_DIR/"
```

## Quick Reference

```bash
# CT logs

# Active enumeration
amass enum -active -d DOMAIN -brute -w WORDLIST -o amass.txt

# Resolve and filter live
cat subdomains.txt | dnsx -resp -o live.txt

# CNAME cloud mapping
while read s; do echo "$s → $(dig +short CNAME $s)"; done < live.txt | grep -v "→ $"

# Takeover check
subjack -w live.txt -t 100 -o takeovers.txt -ssl
nuclei -l live.txt -t ~/nuclei-templates/takeovers/
```
