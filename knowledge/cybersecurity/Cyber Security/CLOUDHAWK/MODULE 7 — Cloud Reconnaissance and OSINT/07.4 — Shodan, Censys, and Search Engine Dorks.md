# 07.4 — Shodan, Censys, and Search Engine Dorks

## What This Note Covers

Using Shodan and Censys to find exposed cloud infrastructure, and Google/Bing dorks targeting cloud-specific assets — exposed admin panels, open registries, unauthenticated databases, and misconfigured cloud services.

---

## Shodan — Internet-Wide Cloud Asset Discovery

Shodan continuously scans the entire internet and indexes banners, certificates, and service responses. It's the fastest way to find exposed cloud services attached to a target organisation.

### Shodan Cloud-Specific Queries

```
# AWS services
org:"Amazon" product:"Docker Registry"
org:"Amazon" port:5000 product:"Docker"
org:"Amazon" port:2375 product:"Docker"     ← exposed Docker API
org:"Amazon" port:9200 product:"Elasticsearch"
org:"Amazon" port:27017 product:"MongoDB"
org:"Amazon" port:6379 product:"Redis"

# Find exposed services by certificate hostname
ssl.cert.subject.cn:"*.targetcompany.com"
ssl:"targetcompany.com" port:443

# Kubernetes API servers
product:"Kubernetes" port:6443
product:"Kubernetes" port:8080           ← unauthenticated HTTP API

# Exposed admin panels in cloud
title:"Grafana" org:"Amazon"
title:"Kibana" org:"Amazon"
title:"Jenkins" org:"Amazon"
title:"Portainer" org:"Amazon"           ← Docker management

# Exposed databases
port:5432 product:"PostgreSQL" org:"Amazon"
port:3306 product:"MySQL" org:"Amazon"
port:27017 product:"MongoDB" has_screenshot:true

# Docker registries
port:5000 "Docker Registry"
port:5000 "/v2/_catalog" 200

# etcd (Kubernetes backing store)
port:2379 product:"etcd"
port:2380 product:"etcd"
```

### Shodan CLI

```bash
# Install
pip3 install shodan

# Configure API key
shodan init YOUR_API_KEY

# Search
shodan search "ssl:targetcompany.com"
shodan search "org:'Target Company, Inc.'"

# Get info about a specific IP
shodan host TARGET_IP

# Download results
shodan download results.json "ssl:targetcompany.com"
shodan parse results.json

# Count results
shodan count "ssl:targetcompany.com"

# Alert — monitor for new assets
shodan alert create "Target Monitor" "net:TARGET_IP_RANGE"
shodan alert list
```

## Censys — Certificate and Port Intelligence

Censys indexes certificates and internet-wide port scans. Strong for certificate-based discovery of cloud assets.

```bash
# Censys CLI
pip3 install censys

# Configure
censys config  # enter API credentials from censys.io

# Search certificates for target domain
censys certificates "parsed.names: targetcompany.com"

# Search IPv4 hosts with specific service
censys ipv4 "protocols: 6443/tcp"   # Kubernetes API
censys ipv4 "protocols: 2375/tcp"   # Docker API (unauthenticated)
censys ipv4 "protocols: 9200/tcp AND tags: cloud"  # Elasticsearch

# Search for specific software
censys ipv4 "software.product: Kubernetes"
censys ipv4 "software.product: etcd"

# Combined: company + exposed service
censys ipv4 "autonomous_system.name: Amazon AND protocols: 9200/tcp"
```

### Censys Web Queries

```
# At search.censys.io

# Find all certificates for a domain including subdomains
parsed.names: targetcompany.com

# Find exposed Kubernetes API servers
services.port: 6443 and services.service_name: KUBERNETES

# Find exposed Docker APIs
services.port: 2375 and services.service_name: DOCKER

# Find cloud assets with specific TLS cert
parsed.subject_dn: "O=Target Company"

# Exposed Redis with no auth
services.port: 6379 and services.redis.info.redis_version: *

# Elasticsearch without auth
services.port: 9200 and services.elasticsearch.cluster_name: *
```

## Combining Tools — Cloud Recon Workflow

```bash
TARGET="targetcompany.com"
TARGET_ORG="Target Company, Inc."  # as it appears in WHOIS/certificates

# Phase 1: Certificate-based discovery
censys certificates "parsed.names: $TARGET" > censys-certs.txt
curl -s "https://crt.sh/?q=%.$TARGET&output=json" | \
  python3 -c "import sys,json; [print(c['name_value']) for c in json.load(sys.stdin)]" \
  > ct-subdomains.txt

# Phase 2: IP range identification
# Get AS number for the company
curl -s "https://ipinfo.io/org?token=TOKEN" | grep "AS"
# Then get all IPs for that ASN
shodan search "asn:ASNNUMBER" --fields ip_str --limit 1000 > ips.txt

# Phase 3: Port and service scanning on discovered IPs
shodan search "org:\"$TARGET_ORG\" port:6443" --fields ip_str,port
shodan search "org:\"$TARGET_ORG\" port:2375" --fields ip_str,port
shodan search "org:\"$TARGET_ORG\" port:9200" --fields ip_str,port
shodan search "ssl:\"$TARGET\"" --fields ip_str,port,hostnames

# Phase 4: For each discovered IP — get full service details
for IP in $(cat ips.txt | head -20); do
  shodan host $IP 2>/dev/null | python3 -c "
import sys
for line in sys.stdin:
    print(line.strip())
" | grep -E "Port|Service|Product|Version|Title"
done

# Phase 5: Correlate with DNS
# What subdomains resolve to the discovered IPs?
while read ip; do
  echo "=== $ip ==="
  host $ip 2>/dev/null | grep "domain name pointer"
done < ips.txt
```

## Exposed Database Discovery and Testing

Shodan is particularly useful for finding internet-exposed databases that should only be accessible internally.

```bash
# Find Redis with no auth (returns "+PONG" to connection)
shodan search "port:6379 +PONG org:Amazon" --fields ip_str

# If found — test connectivity and read keys
redis-cli -h TARGET_IP ping
redis-cli -h TARGET_IP info server
redis-cli -h TARGET_IP keys "*"
redis-cli -h TARGET_IP get "SESSION_KEY"

# Elasticsearch — find and query
curl http://TARGET_IP:9200/
curl http://TARGET_IP:9200/_cat/indices?v
curl http://TARGET_IP:9200/_cat/indices?v | awk '{print $3}' | \
  while read idx; do
    echo "=== $idx ==="
    curl -s "http://TARGET_IP:9200/$idx/_search?size=1" | python3 -m json.tool | head -30
  done

# MongoDB — find and query
mongo --host TARGET_IP --port 27017
> show dbs
> use TARGET_DB
> show collections
> db.users.findOne()
```
