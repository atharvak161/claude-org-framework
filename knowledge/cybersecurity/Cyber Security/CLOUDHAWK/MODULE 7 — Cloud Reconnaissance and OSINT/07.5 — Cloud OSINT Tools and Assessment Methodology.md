# 07.5 — Cloud OSINT Tools and Assessment Methodology

## What This Note Covers

The complete cloud OSINT toolset with install and usage reference, and the systematic reconnaissance methodology that ties Modules 07.1-07.4 together into a structured pre-engagement workflow.

## Core Cloud OSINT Tool Reference

### cloud_enum — Multi-Cloud Asset Discovery

Enumerates resources across AWS, Azure, and GCP simultaneously using keyword-based naming guesses.

```bash
# Install
git clone https://github.com/initstring/cloud_enum
cd cloud_enum
pip3 install -r requirements.txt

# Enumerate all three clouds with a keyword
python3 cloud_enum.py -k targetcompany

# Multiple keywords
python3 cloud_enum.py -k targetcompany -k target-co -k tgtco

# Target specific clouds only
python3 cloud_enum.py -k targetcompany --disable-gcp
python3 cloud_enum.py -k targetcompany --disable-aws --disable-gcp  # Azure only

# Use a custom wordlist (appended to keyword)
python3 cloud_enum.py -k targetcompany -kf custom-suffixes.txt

# What cloud_enum checks:
# AWS: S3 buckets, CloudFront distributions, AWS CodeCommit repos
# Azure: Blob Storage accounts, App Services, Key Vaults, SQL, CosmosDB
# GCP: GCS buckets, App Engine apps, Cloud Functions endpoints
```

### S3Scanner — AWS Bucket Enumeration

```bash
pip3 install s3scanner

# Single bucket check
s3scanner scan --bucket targetcompany-backups

# Bulk from wordlist
s3scanner scan --bucket-file wordlist.txt

# Output: shows permissions level for each bucket
# READ, WRITE, READ_ACP, WRITE_ACP, FULL_CONTROL
# or: NoSuchBucket / AuthorizationHeaderMalformed

# Check with specific AWS profile (for authenticated check)
s3scanner scan --bucket-file wordlist.txt --profile pentest-profile
```

### GCPBucketBrute — GCS Bucket Enumeration

```bash
git clone https://github.com/RhinoSecurityLabs/GCPBucketBrute
pip3 install -r requirements.txt

python3 gcpbucketbrute.py -k targetcompany -u   # unauthenticated
python3 gcpbucketbrute.py -k targetcompany       # authenticated (uses gcloud creds)
python3 gcpbucketbrute.py -k targetcompany -f extra-keywords.txt
```

### CloudBrute — Cloud Infrastructure Brute Force

```bash
git clone https://github.com/0xsha/CloudBrute
cd CloudBrute
go build .

# Enumerate across providers
./CloudBrute -d targetcompany.com -k targetcompany -t 80 -T 10 \
  -w wordlist.txt -c configs/aws.yaml

# Provider-specific configs: aws.yaml, azure.yaml, gcp.yaml
```

### Bucket Finder (Ruby)

```bash
gem install bucket_finder  # or get from GitHub

bucket_finder targetcompany
bucket_finder targetcompany --download  # download public bucket contents
```

## Systematic Cloud Reconnaissance Methodology

### Phase 0 — Scope and Target Definition

Before any tool runs:

```
Define the target:
  - Primary domain: targetcompany.com
  - Known cloud accounts: AWS Account ID (if known), Azure Tenant ID, GCP Org
  - Known subsidiary domains: targetinternational.com, acquiredcompany.com
  - Brands and product names (may have separate cloud accounts)
  - GitHub organisation names
  - DockerHub organisation names

Define your constraints:
  - Is this passive (no active probing) or active?
  - Are you permitted to interact with cloud APIs unauthenticated?
  - Are you permitted to access publicly available resources?
```

### Phase 1 — Passive OSINT (No Target Interaction)

```bash
# CT log enumeration
curl -s "https://crt.sh/?q=%.targetcompany.com&output=json" | \
  python3 -c "import sys,json; [print(c['name_value']) for c in json.load(sys.stdin)]" \
  | sort -u > phase1-ct.txt

# Passive DNS enumeration
subfinder -d targetcompany.com -all -o phase1-subfinder.txt

# GitHub reconnaissance
trufflehog github --org TARGETORG --only-verified
gh repo list TARGETORG --limit 1000 --json name,url

# Shodan passive query (no direct target probing)
shodan search "ssl:targetcompany.com" --fields ip_str,port,hostnames
shodan search "org:'Target Company, Inc.'" --fields ip_str,port,product

# DNS record analysis
dig targetcompany.com ANY
dig targetcompany.com TXT  # reveals SaaS/cloud services via SPF
dig targetcompany.com MX   # reveals email provider

# Job postings analysis
# LinkedIn/Indeed: search "targetcompany" + "AWS"/"Azure"/"GCP"
# Job descriptions reveal exact cloud stack being used
```

### Phase 2 — Active Cloud Enumeration (Interacts with Cloud APIs)

```bash
# Multi-cloud storage enumeration
python3 cloud_enum.py -k targetcompany -k target -k tco
python3 cloud_enum.py -k $(cat related-keywords.txt | tr '\n' ' ' | sed 's/ / -k /g')

# AWS-specific
s3scanner scan --bucket-file wordlist.txt

# Azure-specific
# Check blob storage accounts directly
while read account; do
  status=$(curl -s -o /dev/null -w "%{http_code}" \
    "https://$account.blob.core.windows.net/?comp=list")
  echo "$account: $status"
done < azure-account-names.txt

# GCP-specific
python3 gcpbucketbrute.py -k targetcompany -u

# Subdomain resolution
cat phase1-ct.txt phase1-subfinder.txt | sort -u | \
  dnsx -resp -o phase2-live.txt
```

### Phase 3 — Service Identification

```bash
# HTTP probe all discovered assets
httpx -l phase2-live.txt \
  -sc -title -server -td \
  -o phase3-http.txt

# CNAME mapping for cloud identification
while read sub; do
  cname=$(dig +short CNAME $sub 2>/dev/null | head -1)
  [ -n "$cname" ] && echo "$sub → $cname"
done < phase2-live.txt | tee phase3-cnames.txt

# Identify cloud services from CNAMEs
echo "=== AWS ==="
grep -i "amazonaws\|cloudfront\|execute-api" phase3-cnames.txt

echo "=== Azure ==="
grep -i "azure\|windows.net\|msecnd" phase3-cnames.txt

echo "=== GCP ==="
grep -i "appspot\|run.app\|cloudfunctions\|googleapis" phase3-cnames.txt

# Check for exposed cloud management interfaces
cat phase2-live.txt | \
  httpx -path "/api/v1/namespaces" -sc -o phase3-k8s.txt  # Kubernetes API
cat phase2-live.txt | \
  httpx -path "/v2/_catalog" -sc -o phase3-registry.txt   # Docker Registry
```

### Phase 4 — Vulnerability Identification

```bash
# Subdomain takeover
subjack -w phase2-live.txt -t 100 -timeout 30 -o phase4-takeovers.txt -ssl
nuclei -l phase2-live.txt -t ~/nuclei-templates/takeovers/ -o phase4-nuclei-takeovers.txt

# Public cloud storage
# Anything identified as S3/GCS/Blob in phase 3 — test for public access
grep "amazonaws.com" phase3-cnames.txt | awk -F' → ' '{print $2}' | \
  sed 's/.s3.amazonaws.com//' | \
  while read bucket; do
    echo "Testing: $bucket"
    aws s3 ls s3://$bucket --no-sign-request 2>&1 | head -3
  done

# Exposed admin panels
nuclei -l phase2-live.txt \
  -t ~/nuclei-templates/exposed-panels/ \
  -o phase4-panels.txt
```

### Phase 5 — Intelligence Synthesis

```bash
# Compile findings into structured format
cat > phase5-summary.md << EOF
# Cloud Recon Summary — targetcompany.com

## Discovered Cloud Accounts
- AWS: [Account IDs found, if any]
- Azure: Tenant ID [TENANT_ID]
- GCP: Projects [PROJECT_IDs]

[List from phase3/phase4]

### Private Buckets (enumerated)
[List from cloud_enum]
```
