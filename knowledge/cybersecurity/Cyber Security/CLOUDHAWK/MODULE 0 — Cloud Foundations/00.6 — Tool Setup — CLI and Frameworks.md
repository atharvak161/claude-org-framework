# 00.6 — Tool Setup — CLI Tools and Security Frameworks

## What This Note Covers

The core tools for every cloud security engagement — installation, configuration, and first-use syntax for AWS CLI, Azure CLI, gcloud CLI, and the major cloud security frameworks: ScoutSuite, Prowler, CloudMapper, Pacu, ROADtools, Stormspotter, and SkyArk.

---

## AWS CLI

### What It Is
The official command-line interface for interacting with all AWS services. Every cloud pentester needs this. All AWS API calls can be made through it.

### Install

```bash
# Linux/macOS
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# macOS (Homebrew)
brew install awscli

# Windows
# Download and run: https://awscli.amazonaws.com/AWSCLIV2.msi

# Verify
aws --version
```

### Configure with Credentials

```bash
# Interactive setup (prompts for key, secret, region, output format)
aws configure

# Or set directly (for a specific profile)
aws configure --profile clientname

# Or use environment variables (preferred for automation/testing)
export AWS_ACCESS_KEY_ID=AKIA...
export AWS_SECRET_ACCESS_KEY=...
export AWS_DEFAULT_REGION=us-east-1

# Verify the identity you're operating as
aws sts get-caller-identity
```

### Core Flags

```bash
--profile PROFILE_NAME    # Use a specific configured profile
--region REGION           # Override the default region
--output json|table|text  # Output format
--query 'JMESPath'        # Filter output using JMESPath expressions
--no-sign-request         # Make unauthenticated requests (useful for public resource testing)
```

### Essential First Commands on Engagement

```bash
# Who am I?
aws sts get-caller-identity

# What S3 buckets exist?
aws s3 ls

# What EC2 instances are running? (per region)
aws ec2 describe-instances --region us-east-1 --query 'Reservations[].Instances[].{ID:InstanceId,State:State.Name,IP:PublicIpAddress}'

# What IAM users exist?
aws iam list-users

# What IAM roles exist?
aws iam list-roles

# What policies are attached to current user?
aws iam list-attached-user-policies --user-name USERNAME
```

## Azure CLI (az)

### What It Is
Microsoft's official CLI for managing Azure resources and Azure AD / Entra ID. Equivalent to AWS CLI for the Azure ecosystem.

### Install

```bash
# Linux (Debian/Ubuntu)
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

# macOS
brew install azure-cli

# Windows (winget)
winget install -e --id Microsoft.AzureCLI

# Verify
az --version
```

### Authenticate

```bash
# Interactive login (opens browser)
az login

# Login with service principal (non-interactive, common in testing)
az login --service-principal \
  --username APP_ID \
  --password CLIENT_SECRET \
  --tenant TENANT_ID

# Login with device code (for headless environments)
az login --use-device-code

# Verify current identity
az account show
```

### Core First Commands

```bash
# Current identity and subscription
az account show

# List all subscriptions accessible
az account list --output table

# Switch to a specific subscription
az account set --subscription SUBSCRIPTION_ID

# List all resource groups
az group list --output table

# List all VMs
az vm list --output table

# List all storage accounts
az storage account list --output table

# List all Azure AD users (requires appropriate permissions)
az ad user list --output table
```

## Google Cloud CLI (gcloud)

### What It Is
Google's official CLI for GCP. Manages compute, storage, IAM, Kubernetes, and all other GCP services.

### Install

```bash
# Linux/macOS (interactive installer)
curl https://sdk.cloud.google.com | bash
exec -l $SHELL

# macOS (Homebrew)
brew install --cask google-cloud-sdk

# Verify
gcloud --version
```

### Authenticate

```bash
# Interactive login (opens browser)
gcloud auth login

# Application default credentials (for SDK/API use)
gcloud auth application-default login

# Authenticate as a service account (using key file)
gcloud auth activate-service-account \
  --key-file=service-account-key.json

# Check current auth
gcloud auth list
gcloud config list
```

### Core First Commands

```bash
# Set project
gcloud config set project PROJECT_ID

# List projects accessible
gcloud projects list

# Who am I?
gcloud auth list

# List compute instances
gcloud compute instances list

# List storage buckets
gcloud storage buckets list

# List IAM bindings on a project
gcloud projects get-iam-policy PROJECT_ID

# List service accounts
gcloud iam service-accounts list
```

## ScoutSuite — Multi-Cloud Misconfiguration Scanner

### What It Is
Open-source multi-cloud security auditing tool by NCC Group. Connects to a cloud account, pulls config data from all services, and generates an HTML report of security issues. Supports AWS, Azure, GCP, and others.

### Install

```bash
git clone https://github.com/nccgroup/ScoutSuite
cd ScoutSuite
pip3 install virtualenv
virtualenv -p python3 venv
source venv/bin/activate
pip install -r requirements.txt
```

### Usage

```bash
# AWS (uses current AWS CLI credentials)
python3 scout.py aws

# AWS with a specific profile
python3 scout.py aws --profile clientprofile

# Azure (interactive login)
python3 scout.py azure --cli

# Azure with service principal
python3 scout.py azure --service-principal \
  --client-id CLIENT_ID \
  --client-secret CLIENT_SECRET \
  --tenant-id TENANT_ID

# GCP
python3 scout.py gcp --user-account --project PROJECT_ID

# Output saved to: scoutsuite-report/
# Open: scoutsuite-report/scoutsuite-results/scoutsuite_results_aws-*.js
```

### What to Look For in the Report

ScoutSuite flags findings by severity. Priority review areas:
- IAM: users without MFA, over-permissive policies, unused access keys
- S3: public buckets, buckets without encryption, buckets without logging
- EC2: security groups with `0.0.0.0/0` on sensitive ports, instances without monitoring
- CloudTrail: disabled regions, no log file validation
- RDS: publicly accessible instances, unencrypted storage

## Prowler — AWS/Azure/GCP Compliance Scanner

### What It Is
Open-source cloud security tool focused on compliance and best practices. Checks against CIS Benchmarks, PCI DSS, SOC2, GDPR, and more. More compliance-focused than ScoutSuite.

### Install

```bash
pip3 install prowler

# Or from source
git clone https://github.com/prowler-cloud/prowler
cd prowler
pip3 install -r requirements.txt
```

### Usage

```bash
# Run all checks against AWS (uses current credentials)
prowler aws

# Run against specific AWS services
prowler aws --services s3 iam ec2

# Run against specific compliance framework
prowler aws --compliance cis_1.5_aws

# Output as HTML report
prowler aws -M html

# Azure
prowler azure --az-cli-auth

# GCP
prowler gcp --project-id PROJECT_ID
```

## Pacu — AWS Exploitation Framework

### What It Is
Pacu is the Metasploit for AWS. Open-source AWS exploitation framework developed by Rhino Security Labs. Contains modules for enumeration, privilege escalation, persistence, exfiltration, and more.

### Install

```bash
git clone https://github.com/RhinoSecurityLabs/pacu
cd pacu
pip3 install -r requirements.txt
python3 pacu.py
```

### Basic Usage

```bash
# Start Pacu (creates a session)
python3 pacu.py

# Import AWS keys into the session
Pacu> set_keys

# Run a module — enumerate IAM permissions
Pacu> run iam__enum_permissions

# Run S3 enumeration
Pacu> run s3__enum_bucket_portion

# Run privilege escalation check
Pacu> run iam__privesc_scan

# Show all available modules
Pacu> list

# Show module help
Pacu> help iam__privesc_scan
```

> [!warning] Use Pacu carefully in production
> Some modules take destructive or high-visibility actions. Always read a module's description before running. Use `--dry-run` where available during initial testing.

## ROADtools — Azure AD Enumeration

### What It Is
ROADtools is an Azure AD/Entra ID enumeration and analysis framework. It dumps Azure AD data and provides a graphical interface for exploring users, groups, app registrations, service principals, and relationships.

### Install

```bash
pip3 install roadtools
pip3 install roadrecon
```

### Usage

```bash
# Authenticate (device code flow)
roadrecon auth --device-code

# Gather all Azure AD data
roadrecon gather

# Start the GUI explorer (web interface at localhost:5000)
roadrecon gui
```

ROADtools collects: users, groups, service principals, app registrations, OAuth permission grants, device registrations, role assignments. Extremely useful for mapping privilege escalation paths in Azure AD.

## CloudMapper — AWS Network Visualisation

### What It Is
Open-source tool by Duo Security for visualising AWS account network topology. Helps you understand VPC configurations, security groups, and connectivity at a glance.

### Install

```bash
git clone https://github.com/duo-labs/cloudmapper
cd cloudmapper
pip3 install -r requirements.txt
```

### Usage

```bash
# Collect account data
python3 cloudmapper.py collect --account ACCOUNT_NAME

# Generate the network visualisation
python3 cloudmapper.py prepare --account ACCOUNT_NAME

# Serve the web UI
python3 cloudmapper.py webserver
# Open: http://localhost:8000
```

## Quick Reference — Tool Command Summary

```bash
# INITIAL ENGAGEMENT FLOW:

# 1. Validate credentials
aws sts get-caller-identity              # AWS
az account show                         # Azure
gcloud auth list && gcloud config list  # GCP

# 2. Run automated misconfiguration scan
python3 scout.py aws                    # ScoutSuite
prowler aws                             # Prowler

# 3. Start manual exploitation framework
python3 pacu.py                         # Pacu (AWS)
roadrecon auth --device-code            # ROADtools (Azure)

# 4. Enumerate IAM
aws iam list-users && aws iam list-roles
python3 enumerate-iam.py --access-key AKIA... --secret-key ...

# 5. Check for public resources
aws s3 ls                               # S3 buckets
aws ec2 describe-security-groups --filters "Name=ip-permission.cidr,Values=0.0.0.0/0"
```
