# 01.3 — Subdomain Enumeration

## Why Subdomains Matter

The target's primary domain (`example.com`) is usually hardened — it's the front door. Subdomains are the back windows.

Common findings on subdomains:
- **Staging/dev environments** with no WAF, debug mode on, or outdated software
- **Internal tools** accidentally exposed to the internet (Jira, Confluence, Jenkins, GitLab)
- **Old forgotten apps** that were "temporarily" public three years ago
- **Different attack surfaces** — the main site runs React on nginx; the subdomain runs a PHP app from 2014

> [!tip]
> Bug bounty hunters find the majority of their critical findings on subdomains. The main app is tested constantly. `staging-api.example.com` is not.

---

## Method 1 — Certificate Transparency Logs (Best Passive Source)

Already covered in 01.1. The fastest way to get a comprehensive subdomain list passively.

```bash
# crt.sh — all subdomains with issued certs
curl -s "https://crt.sh/?q=%.example.com&output=json" \
  | jq -r '.[].name_value' \
  | sed 's/\*\.//g' \
  | sort -u \
  > crtsh_subdomains.txt

cat crtsh_subdomains.txt
```

## Method 2 — Subfinder (Fast, Multi-Source)

Subfinder queries passive sources (crt.sh, Shodan, VirusTotal, SecurityTrails, etc.) and returns subdomains without directly touching the target.

```bash
# Install
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest
# Or on Kali:
sudo apt install subfinder
```

```bash
# Basic usage
subfinder -d example.com

# With more sources (requires API keys configured)
subfinder -d example.com -all

# Save output
subfinder -d example.com -o subfinder_results.txt

# Multiple domains
subfinder -dL domains.txt -o subfinder_results.txt

# Silent output (subdomains only, no banner)
subfinder -d example.com -silent

# Show sources found in
subfinder -d example.com -sources
```

### Configuring API Keys (Significantly Increases Results)

```bash
# Config file location
~/.config/subfinder/provider-config.yaml
```

Add keys for: `shodan`, `censys`, `virustotal`, `securitytrails`, `binaryedge`, `urlscan`

Free APIs to get:
- VirusTotal: https://www.virustotal.com/gui/my-apikey
- Shodan: https://account.shodan.io
- URLScan: https://urlscan.io/user/signup

## Method 3 — Amass (Most Comprehensive)

Amass is the most powerful subdomain enumeration tool. It combines passive OSINT with active DNS techniques and builds a relationship graph of discovered assets.

```bash
# Install
sudo apt install amass
# Or:
go install -v github.com/owasp-amass/amass/v4/...@master
```

### Amass Modes

```bash
# Passive only (OSINT sources, no DNS queries to target)
amass enum -passive -d example.com

# Active (includes DNS bruteforcing, permutations)
amass enum -active -d example.com

# Full — passive + active + brute force
amass enum -d example.com -brute -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt
```

### Useful Amass Commands

```bash
# Save output to file
amass enum -d example.com -o amass_results.txt

# Show IP addresses alongside subdomains
amass enum -d example.com -ip

# Multiple domains
amass enum -df domains.txt -o results.txt

# Check Amass database for previously found assets
amass db -d example.com -show

# Visualise the graph (outputs to Maltego-compatible format)
amass viz -d3 -d example.com
```

> [!warning]
> Amass is slow. For large scopes, passive mode can take 30+ minutes. Run it in a tmux session and let it work in the background while you use other tools.

Simple, fast tool by tomnomnom. Great for quick passive results.


## Method 5 — DNS Bruteforcing

The passive methods only find subdomains that are already in public sources. DNS bruteforcing tries to **discover hidden subdomains** by guessing names from a wordlist.

For each word in the wordlist, the tool queries: `word.example.com` — if it resolves, it's a real subdomain.

### gobuster (DNS mode)

```bash
gobuster dns -d example.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -t 50

# Show IPs in output
gobuster dns -d example.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -t 50 --show-ips
```

### ffuf (DNS mode)

```bash
ffuf -u https://FUZZ.example.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -v

# Filter out responses with certain word counts (to remove wildcard DNS false positives)
ffuf -u https://FUZZ.example.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -fw 1
```

### Best Wordlists for DNS Bruteforcing

```bash
# From SecLists (ranked by size/effectiveness)
/usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt      # Fast
/usr/share/seclists/Discovery/DNS/subdomains-top1million-20000.txt     # Balanced
/usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt    # Thorough
/usr/share/seclists/Discovery/DNS/dns-Jhaddix.txt                     # Community favourite
/usr/share/seclists/Discovery/DNS/combined_subdomains.txt              # Comprehensive
```

> [!tip]
> Always start with the 5000-word list for speed. Only move to larger lists if you have time or the target is large.

## Method 6 — Permutation & Alteration

If you find `api.example.com`, there's likely also `api-v2.example.com`, `api-dev.example.com`, `api-staging.example.com`. Permutation tools generate these variations automatically.

### dnsgen

```bash
# Install
pip3 install dnsgen

# Generate permutations from a known subdomain list
cat subfinder_results.txt | dnsgen - | massdns -r /opt/resolvers.txt -t A -o S - > permutation_results.txt
```

### altdns

```bash
# Install
pip3 install altdns

# Generate alterations
altdns -i subfinder_results.txt -o permutations.txt -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt
```

## Method 7 — DNSx (DNS Resolution & Validation)

After you have a large list of potential subdomains, DNSx resolves them all quickly to confirm which ones are real (have valid DNS records).

```bash
# Install
go install -v github.com/projectdiscovery/dnsx/cmd/dnsx@latest

# Resolve a list of subdomains — output only those that resolve
cat all_subdomains.txt | dnsx -silent

# Show A records (IPs)
cat all_subdomains.txt | dnsx -a -resp

# Show CNAME records (useful for subdomain takeover hunting)
cat all_subdomains.txt | dnsx -cname -resp

# Filter by specific record type
cat all_subdomains.txt | dnsx -a -resp-only
```

## Subdomain Takeover

A subdomain takeover occurs when a subdomain's DNS record points to a service (AWS S3 bucket, Heroku app, GitHub Pages, etc.) that **no longer exists** — and an attacker can claim that unclaimed service.

Example:
1. `assets.example.com` → CNAME → `example-assets.s3.amazonaws.com`
2. The S3 bucket `example-assets` is deleted
3. The CNAME record is NOT removed from DNS
4. An attacker creates `example-assets` S3 bucket
5. Now `assets.example.com` serves attacker-controlled content

### How to Find Subdomain Takeover Candidates

```bash
# Check CNAME records for all subdomains
cat all_subdomains.txt | dnsx -cname -resp

# Look for CNAMEs pointing to:
# *.s3.amazonaws.com (S3)
# *.azurewebsites.net (Azure)
# *.github.io (GitHub Pages)
# *.herokuapp.com (Heroku)
# *.netlify.app (Netlify)
# *.vercel.app (Vercel)
```

### subjack (Automated Takeover Check)

```bash
# Install
go install github.com/haccer/subjack@latest

# Check for takeovers
subjack -w subdomains.txt -t 100 -timeout 30 -ssl -c /path/to/fingerprints.json -v
```

> [!tip]
> Subdomain takeover is a common bug bounty finding. Even Low/Medium severity findings are easy wins if you find an unclaimed CNAME. The impact is demonstrated by serving content from the subdomain.

## Full Subdomain Enumeration Workflow

```bash
TARGET=example.com

# Step 1 — Passive sources
subfinder -d $TARGET -silent -o passive_subs.txt
assetfinder --subs-only $TARGET >> passive_subs.txt
curl -s "https://crt.sh/?q=%.$TARGET&output=json" | jq -r '.[].name_value' | sed 's/\*\.//g' >> passive_subs.txt

# Step 2 — Sort and deduplicate
sort -u passive_subs.txt -o passive_subs.txt

# Step 3 — DNS bruteforce
gobuster dns -d $TARGET -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -t 50 \
  | grep "Found:" | awk '{print $2}' > brute_subs.txt

# Step 4 — Combine all results
cat passive_subs.txt brute_subs.txt | sort -u > all_subs.txt

# Step 5 — Resolve to confirm live subdomains
cat all_subs.txt | dnsx -silent > live_subs.txt

echo "Live subdomains found: $(wc -l < live_subs.txt)"

# Step 6 — Port scan live subdomains
nmap -iL live_subs.txt -T4 --top-ports 1000 -oA subdomain_scan
```

## Quick Reference

```bash
# Passive sources
subfinder -d example.com -silent -o subs.txt
assetfinder --subs-only example.com >> subs.txt
curl -s "https://crt.sh/?q=%.example.com&output=json" | jq -r '.[].name_value' | sort -u >> subs.txt

# Active brute force
gobuster dns -d example.com -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-5000.txt -t 50

# Resolve/validate
cat subs.txt | dnsx -silent > live_subs.txt

# CNAME check (takeover hunting)
cat live_subs.txt | dnsx -cname -resp
```
