# 01.2 — Active Recon & Nmap

## What It Is

Active reconnaissance means **directly interacting with the target's systems** — sending packets and analysing responses. Unlike passive recon, this is **detectable**. Your IP will appear in the target's firewall logs and IDS alerts.

This is why you do passive recon first: minimise your active footprint until you understand the target, and only then begin touching systems.

The primary tool for active recon is **Nmap** — the Network Mapper. It is the industry standard for host discovery, port scanning, service version detection, and OS fingerprinting.

---

## How TCP Works (Essential Background)

Before scanning, you need to understand the TCP three-way handshake, because Nmap scan types are defined by how they manipulate it.

```
Client (You)          Server (Target)
     │                      │
     │──── SYN ────────────►│    "I want to connect"
     │◄─── SYN-ACK ─────────│    "OK, I'm listening"
     │──── ACK ────────────►│    "Connection established"
     │                      │
     │    [data exchange]    │
     │                      │
     │──── FIN ────────────►│    "I'm done"
     │◄─── ACK ─────────────│    "Acknowledged"
```

- **Open port**: completes the handshake (SYN → SYN-ACK → ACK)
- **Closed port**: RST is returned instead of SYN-ACK
- **Filtered port**: no response (firewall dropped the packet) or ICMP unreachable

## Nmap — Core Concepts

### Install / Verify

```bash
# Already on Kali. Verify:
nmap --version
```

### Basic Syntax

```bash
nmap [scan type] [options] [target]
```

Target can be:
```bash
nmap 192.168.1.1          # Single IP
nmap 192.168.1.1-254      # Range
nmap 192.168.1.0/24       # CIDR subnet
nmap scanme.nmap.org      # Hostname
nmap -iL targets.txt      # From file
```

## Scan Types

### -sS — SYN Scan (Default, Stealth)

```bash
sudo nmap -sS 192.168.1.1
```

How it works:
1. Send SYN
2. Receive SYN-ACK (port open) or RST (port closed) or nothing (filtered)
3. **Send RST immediately** — never complete the handshake

Why it's stealthy: the full TCP connection is never established, so many older logging systems don't record it. Modern IDS will still catch it, but it's the least noisy scan type.

Requires root/sudo (needs raw socket access).

### -sT — TCP Connect Scan

```bash
nmap -sT 192.168.1.1
```

Completes the full three-way handshake. Used when you don't have root privileges. **Slower and more detectable** — connection attempts are fully logged by the target.

### -sU — UDP Scan

```bash
sudo nmap -sU 192.168.1.1
```

Scans UDP ports. Much slower than TCP (no handshake — relies on ICMP port unreachable responses or service-specific probes).

Critical services run on UDP:
- DNS (53)
- DHCP (67/68)
- SNMP (161)
- TFTP (69)
- NTP (123)
- LDAP (389)

> [!tip]
> UDP scanning is slow. Target only key ports unless you have time: `sudo nmap -sU -p 53,67,68,69,123,161,162,389,500 192.168.1.1`

### -sN / -sF / -sX — NULL, FIN, Xmas Scans

```bash
sudo nmap -sN 192.168.1.1   # No flags set
sudo nmap -sF 192.168.1.1   # FIN flag only
sudo nmap -sX 192.168.1.1   # FIN + PSH + URG (lit up like a Christmas tree)
```

These exploit RFC 793 behaviour: if a port is **closed**, the target sends RST. If **open or filtered**, no response.

Used for firewall evasion against stateless packet filters. **Do not work against Windows** — Windows sends RST for all ports regardless.

### -sA — ACK Scan

```bash
sudo nmap -sA 192.168.1.1
```

Not for finding open ports — for **mapping firewall rules**. 
- RST = port is **unfiltered** (firewall lets ACK through)
- No response = port is **filtered** (firewall is blocking it)

Useful for understanding firewall topology.

### -sn — Ping Scan (Host Discovery Only, No Port Scan)

```bash
nmap -sn 192.168.1.0/24
```

Discovers which hosts are alive on a network. Does not scan ports. Fast — use this first on large subnets.

## Essential Options

### Port Selection

```bash
-p 80                    # Single port
-p 80,443,8080           # Multiple ports
-p 1-1000                # Range
-p-                      # All 65535 ports
--top-ports 100          # Top 100 most common ports
--top-ports 1000         # Top 1000 (default if no -p given)
-F                       # Fast mode — top 100 ports
```

### Timing Templates (-T)

Controls speed vs stealth trade-off:

| Template | Name | Description |
|---|---|---|
| -T0 | Paranoid | Very slow — evades IDS, 5 min between probes |
| -T1 | Sneaky | Slow — IDS evasion |
| -T2 | Polite | Slow — reduces bandwidth load |
| -T3 | Normal | **Default** — balanced |
| -T4 | Aggressive | Fast — assumes good network |
| -T5 | Insane | Very fast — likely to miss results |

For labs: `-T4` (fast, reliable)  
For real engagements: `-T2` or `-T3` (quieter)

### Service/Version Detection

```bash
-sV                      # Detect service versions
-sV --version-intensity 9    # Maximum version detection effort
```

Output example:
```
80/tcp  open  http     Apache httpd 2.4.51 ((Ubuntu))
22/tcp  open  ssh      OpenSSH 7.6p1 Ubuntu 4ubuntu0.7
443/tcp open  ssl/http nginx 1.18.0
```

Service versions → search for CVEs directly.

### OS Detection

```bash
-O                       # OS fingerprinting
--osscan-guess           # Guess OS even when not certain
```

Nmap analyses TCP/IP stack behaviour (TTL, window sizes, packet responses) to guess the OS. Requires root.

### Aggressive Scan (-A)

```bash
sudo nmap -A 192.168.1.1
```

Equivalent to: `-sV -O --traceroute` + NSE default scripts. Noisy but comprehensive. Good for labs.

### Output Formats

```bash
-oN output.txt           # Normal (human-readable)
-oX output.xml           # XML (for import into Metasploit, etc.)
-oG output.gnmap         # Grepable (easy to parse with grep/awk)
-oA output              # All three formats simultaneously (recommended)
```

> [!tip]
> **Always save output.** `nmap -oA scan_results <target>` creates three files. You will reference scan results constantly during an engagement. Losing them wastes time.

### Verbosity

```bash
-v     # Verbose — shows results as they arrive
-vv    # Very verbose
-d     # Debug output
```

## NSE — Nmap Scripting Engine

NSE is one of Nmap's most powerful features. Scripts extend Nmap's capabilities to:
- Detect specific vulnerabilities
- Enumerate services (SMB shares, SNMP, DNS)
- Brute-force authentication
- Fingerprint web applications

Scripts are located at: `/usr/share/nmap/scripts/`

```bash
ls /usr/share/nmap/scripts/ | wc -l    # ~600+ scripts
ls /usr/share/nmap/scripts/ | grep smb # SMB-related scripts
```

### Script Categories

| Category | Purpose |
|---|---|
| `default` | Safe, useful — run automatically with -A |
| `safe` | No harmful side effects |
| `discovery` | Enumerate services and hosts |
| `vuln` | Detect known vulnerabilities |
| `exploit` | Actively exploit vulnerabilities (use carefully) |
| `auth` | Test authentication (brute force) |
| `intrusive` | Potentially harmful — may crash services |
| `malware` | Detect malware backdoors |
| `broadcast` | Discover hosts via broadcast |

### Running Scripts

```bash
# Run default scripts
nmap -sC 192.168.1.1

# Run specific script
nmap --script=http-title 192.168.1.1

# Run multiple scripts
nmap --script=http-title,http-headers 192.168.1.1

# Run all scripts in a category
nmap --script=vuln 192.168.1.1

# Run scripts matching a pattern
nmap --script="smb*" 192.168.1.1
```

### High-Value NSE Scripts

#### SMB Enumeration
```bash
# Enumerate SMB shares, users, OS info
nmap --script=smb-enum-shares,smb-enum-users,smb-os-discovery -p 445 192.168.1.1

# Check for EternalBlue (MS17-010)
nmap --script=smb-vuln-ms17-010 -p 445 192.168.1.1

# Check SMB signing (relevant for relay attacks)
nmap --script=smb-security-mode -p 445 192.168.1.1
```

#### HTTP Enumeration
```bash
# Get page title (quick overview of web services)
nmap --script=http-title -p 80,443,8080,8443 192.168.1.0/24

# Enumerate HTTP methods allowed
nmap --script=http-methods -p 80,443 192.168.1.1

# Find interesting directories
nmap --script=http-enum -p 80,443 192.168.1.1

# Check for default credentials on web logins
nmap --script=http-default-accounts -p 80,443 192.168.1.1
```

#### SSL/TLS Analysis
```bash
# Check supported SSL/TLS ciphers and versions (finds SSLv3, weak ciphers)
nmap --script=ssl-enum-ciphers -p 443 192.168.1.1

# Check SSL certificate details
nmap --script=ssl-cert -p 443 192.168.1.1

# Check for Heartbleed (CVE-2014-0160)
nmap --script=ssl-heartbleed -p 443 192.168.1.1
```

#### FTP
```bash
# Check for anonymous FTP login
nmap --script=ftp-anon -p 21 192.168.1.1

# Check FTP bounce attack
nmap --script=ftp-bounce -p 21 192.168.1.1
```

#### DNS
```bash
# DNS zone transfer
nmap --script=dns-zone-transfer --script-args dns-zone-transfer.domain=example.com -p 53 ns1.example.com

# DNS brute force subdomains
nmap --script=dns-brute example.com
```

#### Vulnerability Detection
```bash
# Run all vuln scripts (noisy but comprehensive)
sudo nmap --script=vuln -p- 192.168.1.1

# Specific CVEs
nmap --script=smb-vuln-ms17-010 -p 445 192.168.1.1  # EternalBlue
nmap --script=http-shellshock -p 80 192.168.1.1       # Shellshock
nmap --script=ssl-heartbleed -p 443 192.168.1.1       # Heartbleed
```

## Practical Scan Workflow

This is the standard scanning workflow for an engagement. Run these in order.

### Phase 1 — Host Discovery (Which Hosts Are Alive)

```bash
# Ping sweep — find live hosts
sudo nmap -sn 192.168.1.0/24 -oA host_discovery

# Extract live IPs
grep "Nmap scan report" host_discovery.nmap | awk '{print $5}' > live_hosts.txt
cat live_hosts.txt
```

### Phase 2 — Quick Port Scan (Top 1000 Ports)

```bash
sudo nmap -sS -T4 --top-ports 1000 -iL live_hosts.txt -oA quick_scan
```

### Phase 3 — Full Port Scan (All 65535 Ports)

```bash
# Run in background — this takes time
sudo nmap -sS -T4 -p- -iL live_hosts.txt -oA full_scan
```

### Phase 4 — Service/Version + Default Scripts on Open Ports

```bash
# First, extract open ports from full scan
grep "open" full_scan.nmap | awk -F'/' '{print $1}' | sort -u | paste -sd, > open_ports.txt

# Run version detection + default scripts on those ports only
sudo nmap -sS -sV -sC -T4 -p $(cat open_ports.txt) -iL live_hosts.txt -oA service_scan
```

### Phase 5 — Targeted NSE Scripts Based on Open Services

```bash
# If port 445 is open:
sudo nmap --script=smb-enum-shares,smb-enum-users,smb-vuln-ms17-010 -p 445 192.168.1.1

# If port 443 is open:
sudo nmap --script=ssl-enum-ciphers,ssl-cert -p 443 192.168.1.1

# If port 80 is open:
sudo nmap --script=http-title,http-enum,http-methods -p 80 192.168.1.1
```

## Masscan — High-Speed Port Scanner

Masscan is Nmap's speed-oriented cousin. It can scan the **entire internet's port space in under 6 minutes** (at 10 million packets/second). For large internal networks, it's much faster than Nmap.

```bash
# Install
sudo apt install masscan

# Scan a subnet for port 80 and 443
sudo masscan 192.168.1.0/24 -p 80,443 --rate=1000

# Scan all ports on a host
sudo masscan 192.168.1.1 -p0-65535 --rate=10000

# Save output
sudo masscan 192.168.1.0/24 -p0-65535 --rate=5000 -oL masscan_results.txt
```

> [!warning]
> Masscan at high rates can **crash routers and firewalls**. Keep `--rate` below 1000 on client networks. On your own lab, 10000+ is fine.

Workflow: Use Masscan for fast port discovery, then pipe results to Nmap for service detection:

```bash
# Find open ports fast with Masscan
sudo masscan 192.168.1.0/24 -p0-65535 --rate=5000 -oG masscan_out.txt

# Extract IPs and ports
grep "open" masscan_out.txt | awk '{print $4":"$3}' | sed 's/\/tcp//'

# Then run Nmap on those specific ports for proper detection
sudo nmap -sV -sC -p <open_ports> <IPs>
```

## Netdiscover — ARP-Based Host Discovery

For internal networks, ARP is more reliable than ICMP for finding hosts (firewalls block pings, not ARP).

```bash
sudo netdiscover -r 192.168.1.0/24      # Scan subnet
sudo netdiscover -i eth0                 # Passive — just listen for ARP traffic
```

Output shows: IP, MAC address, Vendor (from MAC OUI lookup). The vendor tells you a lot about a device (e.g., "Apple" = MacBook, "Fortinet" = firewall).

## Quick Reference

```bash
# Host discovery
sudo nmap -sn 192.168.1.0/24

# Quick top-1000 port scan
sudo nmap -sS -T4 --top-ports 1000 192.168.1.1

# Full port scan
sudo nmap -sS -T4 -p- 192.168.1.1

# Service + version + default scripts
sudo nmap -sS -sV -sC -T4 192.168.1.1

# All out aggressive
sudo nmap -A -T4 192.168.1.1

# UDP scan (key ports)
sudo nmap -sU -p 53,67,68,69,123,161 192.168.1.1

# Vuln scripts
sudo nmap --script=vuln 192.168.1.1

# SMB vuln check
sudo nmap --script=smb-vuln-ms17-010 -p 445 192.168.1.1

# Save all formats
sudo nmap -A -p- -oA full_results 192.168.1.1
```

| Flag | Meaning |
|---|---|
| -sS | SYN scan (stealth, default) |
| -sU | UDP scan |
| -sV | Service version detection |
| -sC | Default NSE scripts |
| -A | Aggressive (-sV -O --traceroute -sC) |
| -p- | All 65535 ports |
| -T4 | Fast timing |
| -oA | Save all output formats |
| -iL | Read targets from file |
| --script= | Run specific NSE script(s) |
