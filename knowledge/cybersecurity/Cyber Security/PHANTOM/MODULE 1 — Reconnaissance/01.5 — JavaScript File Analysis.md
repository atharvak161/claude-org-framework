# 01.5 - JavaScript File Analysis

## Why JS Files Are High-Yield Recon

Modern web applications are largely JavaScript-driven. The JS bundles browsers download contain far more than the server intentionally exposes: hidden API endpoints, hardcoded secrets, internal hostnames, authentication logic, and third-party integrations. This step belongs in every web recon workflow immediately after directory brute-forcing and routinely finds things no scanner catches.

---

## Step 1 - Discover and Download JS Files

Install the discovery tools first:

    go install github.com/lc/gau/v2/cmd/gau@latest
    go install github.com/tomnomnom/waybackurls@latest
    go install github.com/hakluke/hakrawler@latest

Run discovery:

    TARGET="target.com"
    TARGET_URL="https://$TARGET"
    
    echo "$TARGET" | gau --blacklist png,jpg,gif,css | grep "\.js$" | sort -u > js_urls.txt
    echo "$TARGET" | waybackurls | grep "\.js$" | sort -u >> js_urls.txt
    echo "$TARGET_URL" | hakrawler -js -depth 3 | sort -u >> js_urls.txt
    curl -s "$TARGET_URL" | grep -oE 'src="[^"]+\.js[^"]*"' | cut -d'"' -f2 >> js_urls.txt
    
    sort -u js_urls.txt -o js_urls.txt
    echo "[*] Found $(wc -l < js_urls.txt) unique JS URLs"
    
    mkdir -p js_files/
    wget -i js_urls.txt -P js_files/ -q 2>/dev/null

## LinkFinder - Extract Endpoints

Install: pip3 install linkfinder

Usage:

    # Single file
    python3 linkfinder.py -i js_files/main.js -o cli
    
    # URL directly
    python3 linkfinder.py -i https://target.com/static/js/main.chunk.js -o cli
    
    # Entire directory
    python3 linkfinder.py -i js_files/ -o cli | sort -u > endpoints.txt
    
    # Crawl page + all linked JS
    python3 linkfinder.py -i https://target.com -d -o cli
    
    # HTML report
    python3 linkfinder.py -i js_files/ -o results.html
    
    # Filter high-value paths
    cat endpoints.txt | grep -E "^/api|/admin|/internal|/debug|/v[0-9]"

Example output from a React bundle:

    /api/v1/users
    /api/v1/admin/impersonate        (not in the UI)
    /api/v1/admin/delete-user
    /api/v2/internal/debug
    https://staging-api.target.com/v1/
    /graphql
    /api/v1/export?format=csv&all=true

Every /admin and /internal path is a testing target. Every staging URL is a less-hardened copy of production.

## SecretFinder - Extract Hardcoded Secrets

Install:

    git clone https://github.com/m4ll0k/SecretFinder
    cd SecretFinder && pip3 install -r requirements.txt

Usage:

    # Single URL
    python3 SecretFinder.py -i https://target.com/static/js/main.chunk.js -o cli
    
    # All JS files
    for f in js_files/*.js; do
        echo "=== $f ==="
        python3 SecretFinder.py -i "$f" -o cli 2>/dev/null
    done | tee secrets_found.txt
    
    # Page + all linked JS
    python3 SecretFinder.py -i https://target.com -o cli -p
    
    # HTML report
    python3 SecretFinder.py -i js_files/ -o report.html

Detected patterns include: Google API Keys (AIzaSy...), AWS Access Keys (AKIA...), Stripe secrets (sk_live_...), GitHub tokens (ghp_...), JWTs, private keys, Firebase URLs, S3 bucket URLs, Basic Auth URLs, Sentry DSNs.

NOTE: SecretFinder produces false positives. Manually verify each finding before reporting. Confirm keys are active with a read-only API call where possible. Never test a payment key against live transactions.

## Manual Grep Patterns

These catch secrets that SecretFinder misses:

    cd js_files/
    
    # Generic API keys and tokens
    grep -rE "(api[_-]?key|apikey|api[_-]?token)\s*[:=]\s*['\"][^'\"]{8,}" . --include="*.js"
    grep -rE "(secret|private[_-]?key|client[_-]?secret)\s*[:=]\s*['\"][^'\"]{8,}" . --include="*.js"
    grep -rE "(password|passwd|pwd)\s*[:=]\s*['\"][^'\"]{4,}" . --include="*.js"
    grep -rE "(access[_-]?token|auth[_-]?token|bearer)\s*[:=]\s*['\"][^'\"]{8,}" . --include="*.js"
    
    # AWS credentials
    grep -rE "AKIA[0-9A-Z]{16}" . --include="*.js"
    
    # Firebase
    grep -rE "apiKey\s*:\s*['\"]AIza[^'\"]{35}" . --include="*.js"
    grep -rE "firebaseio\.com" . --include="*.js"
    
    # Internal/staging URLs
    grep -rE "https?://(staging|stage|dev|test|uat|qa|internal|corp|admin)\." . --include="*.js"
    grep -rE "localhost|127\.0\.0\.1|192\.168\.|10\.\d+\.\d+\.\d+" . --include="*.js"
    
    # S3 buckets
    grep -rE "s3\.amazonaws\.com/[a-z0-9\-]+" . --include="*.js"
    grep -rE "['\"][a-z0-9\-]+\.s3\.amazonaws\.com['\"]" . --include="*.js"
    
    # OAuth client IDs (reveals identity provider)
    grep -rE "(client[_-]?id|client[_-]?secret|oauth)\s*[:=]\s*['\"][^'\"]{8,}" . --include="*.js"
    
    # Sentry DSN (reveals org/project structure)
    grep -rE "https://[a-f0-9]{32}@[a-z0-9]+\.ingest\.sentry\.io" . --include="*.js"
    
    # JWT secrets left in config
    grep -rE "(jwt[_-]?secret|signing[_-]?key)\s*[:=]\s*['\"][^'\"]{4,}" . --include="*.js"

## Handling Minified and Bundled JS

Modern apps (React, Vue, Angular, Next.js) compile all JS into minified bundles. These look like unreadable one-liners but contain all original strings.

Beautify before manual analysis:

    # Install: npm install -g js-beautify
    js-beautify js_files/main.chunk.js > js_files/main_pretty.js
    
    # Grep the readable version
    grep -n "apiKey\|secret\|password\|internal" js_files/main_pretty.js
    
    # Alternative: prettier
    npx prettier --parser babel js_files/main.chunk.js > js_files/main_pretty.js

## Source Maps - The Highest-Yield Finding

Source maps (.js.map files) are generated during compilation and contain the original un-minified source code. Developers regularly forget to exclude them from production builds. When exposed they hand you the complete original frontend source.

Check every discovered JS file for a corresponding .map file:

    while IFS= read -r url; do
        map_url="${url}.map"
        status=$(curl -s -o /dev/null -w "%{http_code}" "$map_url")
        if [ "$status" = "200" ]; then
            echo "[!] SOURCE MAP EXPOSED: $map_url"
            curl -s "$map_url" -o "js_files/$(basename $url).map"
        fi
    done < js_urls.txt

Extract the original source files from a .map:

    python3 - << 'EOF'
    import json, os
    
    with open("js_files/main.chunk.js.map") as f:
        sm = json.load(f)
    
    sources = sm.get("sources", [])
    contents = sm.get("sourcesContent", [None] * len(sources))
    
    for source, content in zip(sources, contents):
        if not content:
            continue
        clean = source.replace("webpack://", "").replace("../", "").lstrip("/")
        path = os.path.join("sourcemap_output/", clean)
        os.makedirs(os.path.dirname(path), exist_ok=True)
        with open(path, "w") as out:
            out.write(content)
        print(f"Extracted: {path}")
    EOF
    
    # Grep the reconstructed source tree
    grep -rE "apiKey|password|secret|token|internal" sourcemap_output/
    grep -rE "AKIA[0-9A-Z]{16}" sourcemap_output/

NOTE: A source map is full source code disclosure. In a report, treat it as a Critical information disclosure finding if the app contains business logic, algorithms, or credentials in the frontend code.

## truffleHog - Git History Scanning

If the target has public GitHub repositories, truffleHog scans the entire commit history for secrets including those added and then deleted (still in git history).

Install:

    curl -sSfL https://raw.githubusercontent.com/trufflesecurity/trufflehog/main/scripts/install.sh \
      | sh -s -- -b /usr/local/bin

Usage:

    # Scan a public GitHub repo
    trufflehog github --repo https://github.com/target-org/target-app
    
    # Verified secrets only (fewer false positives)
    trufflehog github --repo https://github.com/target-org/target-app --only-verified
    
    # Scan an entire organisation
    trufflehog github --org target-org --only-verified
    
    # Scan a local clone
    git clone https://github.com/target-org/target-app /tmp/target_app
    trufflehog git file:///tmp/target_app --only-verified
    
    # JSON output
    trufflehog github --repo https://github.com/target-org/target-app --json | jq .

Finds in git history: AWS keys committed then deleted, database passwords in connection strings, SSH private keys accidentally committed, .env files committed and removed, Stripe/Twilio/SendGrid keys rotated after accidental exposure.

## gitleaks - Alternative Git Scanner

Install: sudo apt install gitleaks

Usage:

    # Scan a repository
    gitleaks detect --source /tmp/target_app
    
    # JSON report
    gitleaks detect --source /tmp/target_app --report-format json --report-path leaks.json
    
    # Recent commits only
    gitleaks detect --source /tmp/target_app --log-opts "--since='30 days ago'"

## Complete JS Recon Workflow

    TARGET="target.com"
    TARGET_URL="https://$TARGET"
    
    # 1 - Discover JS URLs
    echo "$TARGET" | gau --blacklist png,jpg,gif,css | grep "\.js$" >> js_urls.txt
    echo "$TARGET" | waybackurls | grep "\.js$" >> js_urls.txt
    echo "$TARGET_URL" | hakrawler -js -depth 3 >> js_urls.txt
    sort -u js_urls.txt -o js_urls.txt
    
    # 2 - Download
    mkdir -p js_files/
    wget -i js_urls.txt -P js_files/ -q 2>/dev/null
    
    # 3 - Extract endpoints
    python3 linkfinder.py -i js_files/ -o cli 2>/dev/null | sort -u > endpoints.txt
    grep -E "^/api|/admin|/internal|/debug" endpoints.txt > interesting_endpoints.txt
    
    # 4 - Hunt secrets
    for f in js_files/*.js; do
        python3 SecretFinder.py -i "$f" -o cli 2>/dev/null
    done | sort -u > secrets_found.txt
    
    # 5 - Manual high-value grep
    grep -rE "AKIA[0-9A-Z]{16}" js_files/ > aws_keys.txt
    grep -rE "https?://(staging|dev|internal)\." js_files/ > internal_urls.txt
    grep -rE "\.s3\.amazonaws\.com" js_files/ > s3_buckets.txt
    
    # 6 - Check for source maps
    while IFS= read -r url; do
        status=$(curl -s -o /dev/null -w "%{http_code}" "${url}.map")
        [ "$status" = "200" ] && echo "[!] SOURCE MAP EXPOSED: ${url}.map"
    done < js_urls.txt
    
    # 7 - Summary
    echo "JS files:              $(ls js_files/*.js 2>/dev/null | wc -l)"
    echo "Total endpoints:       $(wc -l < endpoints.txt)"
    echo "Interesting endpoints: $(wc -l < interesting_endpoints.txt)"
    echo "Potential secrets:     $(wc -l < secrets_found.txt)"
    echo "AWS keys:              $(wc -l < aws_keys.txt)"
    echo "Internal URLs:         $(wc -l < internal_urls.txt)"
